# Quickstart: Kindergarten Program Package

## Purpose

Use this guide when moving from kindergarten packet artifacts to the assembled kindergarten program layer.

## Use Order

1. Read the [feature spec](/Volumes/data/development/oc/specs/003-kindergarten-program-package/spec.md).
2. Read the [data model](/Volumes/data/development/oc/specs/003-kindergarten-program-package/data-model.md).
3. Review the [classroom materials baseline](/Volumes/data/development/oc/research/14-production-templates/classroom-materials-baseline.md).
4. Confirm the source kindergarten packets exist and are packet-complete.
5. Draft teacher-use artifacts before attempting textbook/workbook/program assembly.

## Start Conditions

Before creating program artifacts, confirm that the source packet includes:

- unit
- lesson
- standards map
- review record
- support-material baseline

If any of those are missing, the packet should not be treated as ready for program assembly.

## First Recommended Outputs

For one kindergarten packet, create in this order:

1. lesson narrative
2. classroom task sheet
3. answer key or exemplar
4. scoring guidance
5. home-practice item if appropriate

Only after those exist should the packet be considered eligible for textbook/workbook/teacher-edition assembly work.

## Current Repository Reality

The repository has many kindergarten packets that are structurally governed drafts.

It does not yet broadly include:

- lesson narratives
- classroom assignments
- homework or home-practice sets
- answer keys
- grading keys
- quiz/test sets

That missing layer is the target of this feature.
