# Quickstart: Spec-Driven Learning System

## Purpose

Help contributors start work in the hard-pivot repository direction without falling back into curriculum-product-first authoring.

## Steps

1. Read the constitution at [constitution.md](/Volumes/data/development/oc/.specify/memory/constitution.md).
2. Read this spec package under [specs/007-spec-driven-learning-system/](/Volumes/data/development/oc/specs/007-spec-driven-learning-system/spec.md).
3. Identify the domain slice you are working in.
4. Define or revise:
   - capability nodes
   - dependencies
   - mastery criteria
   - verification gates
   - failure modes
   - intervention rules
5. Only after that, create downstream artifacts such as lessons, items, dashboards, or software.
6. Preserve explicit draft and review state.

## Anti-Patterns

Do not start by:

- generating full-year curriculum volumes
- expanding grade-level product packaging
- creating assessments without explicit node/gate alignment
- treating standards lists as a substitute for capability modeling

## First Proof-Of-Method Target

Use the mathematics domain, with the current downstream slice rooted in:

- fractions
- signed numbers
- equation solving

The first fully specified node target is unlike-denominator fraction addition and subtraction.
