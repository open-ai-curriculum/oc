# Kindergarten ELA Text Architecture

## Purpose

Define the text families and text-selection architecture for the kindergarten ELA program.

## Core Position

Kindergarten ELA should be built around a deliberate mix of teacher-mediated and student-accessible texts.

The program should not assume that kindergarten students independently read the core instructional texts for every lesson. Much of the strongest kindergarten literacy work comes through shared and interactive text experiences.

## Required Text Families

### 1. Interactive Read-Aloud Texts

Role:

- build listening comprehension
- develop oral language and vocabulary
- support knowledge building
- model text talk and comprehension routines

Expected use:

- daily or near-daily in most modules
- often above independent student reading level
- always teacher-mediated

### 2. Shared Reading Texts

Role:

- support print concepts
- build concept of word
- allow choral rereading and pointing to print
- connect oral language to visible text

Expected use:

- heavily in Modules 1 through 4
- continued as fluency and review support later in the year

Examples:

- patterned charts
- big-book style selections
- song and poem charts
- short class-created shared texts

### 3. Phonological-Awareness Oral Texts

Role:

- support rhyme, syllables, onset-rime, blending, and segmenting
- keep sound work playful and oral

Expected use:

- especially strong in Modules 1 through 3
- spiraled through Module 6 and beyond

Examples:

- chants
- songs
- nursery-rhyme-like language
- oral word lists

### 4. Alphabet And Sound-Pattern Assets

Role:

- support letter recognition, sound association, and print attention

Expected use:

- Modules 2 through 6 especially

Examples:

- alphabet cards
- sound-picture cards
- wall cards
- student reference strips

### 5. Decodable And Controlled Texts

Role:

- support early application of taught sound-spelling patterns
- build confidence with simple connected text

Expected use:

- introduced cautiously in Modules 6 through 9
- never treated as the only valuable text family

Selection rule:

- decodable texts should align to taught patterns without becoming semantically empty or developmentally awkward

### 6. Informational Text Sets

Role:

- build knowledge and topic vocabulary
- support asking and answering questions
- provide content for speaking, drawing, and writing

Expected use:

- concentrated in Modules 5, 7, and 8

Examples:

- short informational books
- image-rich articles
- teacher-made topic pages
- photo-based observation texts

### 7. Picture And Conversation Prompts

Role:

- support oral language, vocabulary, sequencing, and writing rehearsal
- provide access for students who are not yet independently reading text

Expected use:

- across all modules

Examples:

- sequence cards
- scene cards
- vocabulary images
- oral-language discussion boards

### 8. Home Reading And Family-Connection Assets

Role:

- connect classroom literacy routines to home language and family interaction
- support repeated oral reading, story retell, and talk at home

Expected use:

- weekly or module-end distribution where appropriate

## Text-Set Design Rules

- every module should include a coherent text set, not random isolated titles
- interactive read-alouds should build knowledge and vocabulary over time
- shared reading texts should be reusable and visually clear
- decodable texts should be tightly aligned to taught patterns
- informational texts should use strong visuals and concept clarity
- text families should reflect varied identities and avoid narrow or exclusionary framing

## Rights And Provenance Strategy

The repository should treat text sourcing as a major commercialization requirement.

Text options may include:

- original repository-authored texts
- public-domain texts where appropriate
- licensed texts
- teacher-created shared texts and charts

The commercialization build should not assume that any externally sourced text can simply be copied into final products without rights review.

## Accessibility And Multilingual Support

- all text families should allow picture, oral, and gesture support
- complex comprehension work should not depend on independent decoding
- family-facing assets should be designed for translation and multilingual use where possible
- typography, spacing, and page design should support early learners and accessible print use

## Text Architecture By Module

| Module | Primary Text Families |
| --- | --- |
| Module 1 | interactive read-aloud, shared reading, picture prompts |
| Module 2 | oral sound-play texts, songs/chants, alphabet assets, shared reading |
| Module 3 | interactive read-aloud, shared reading, story sequence cards |
| Module 4 | shared reading, alphabet assets, controlled sentence strips |
| Module 5 | informational read-aloud, image-rich informational texts, picture prompts |
| Module 6 | shared reading, decodables, sound-pattern assets |
| Module 7 | narrative and informational anchor texts, picture prompts, writing mentor texts |
| Module 8 | knowledge-building text sets, shared and controlled texts, response prompts |
| Module 9 | mixed review set across narrative, informational, shared, and controlled texts |

## Current Gap

The repository does not yet have a finalized kindergarten ELA text set. This document defines the architecture that text selection and text authoring should follow.
