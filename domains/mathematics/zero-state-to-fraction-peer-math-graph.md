# Zero-State To Fraction-Peer Math Graph

## Purpose

Define the broader early-math knowledge graph that reaches the rough peer level of the current fraction-readiness and early fraction segment.

This document is different from the narrow prerequisite spine in [zero-state-to-f1-spine.md](/Volumes/data/development/oc/domains/mathematics/zero-state-to-f1-spine.md).

That document answers:

- what must exist before `F1` is legitimate

This document answers:

- what other mathematical branches should exist at comparable developmental depth, even when they do not yet connect directly into the current fraction segment

## Design Rule

The repository should not confuse:

- a valid prerequisite path

with:

- a sufficiently broad early-math knowledge graph

Both are required.

## Graph Scope

The peer-level target here is roughly:

- zero-state and pre-counting readiness
- pre-kindergarten and kindergarten foundational numeracy
- early Grade 1 whole-number structure and operations
- the fraction-readiness and early fraction branch already underway

## Core Branches

### A. Zero-State And Pre-Counting Foundations

These nodes remain shared infrastructure for all later branches.

- `Z1` Attention to quantity
- `Z2` Match and sort objects
- `Z3` Pattern and sequence readiness

### B. Counting And Cardinality Branch

This branch is already partially implemented and remains a shared prerequisite branch.

- `C1` Stable counting sequence
- `C2` One-to-one correspondence
- `C3` Cardinality
- `C4` Compare small quantities
- `C5` Numeral recognition and writing readiness
- `C6` Quantity-symbol correspondence

### C. Whole-Number Composition And Operations Branch

This branch is not only a prerequisite to fractions. It is also a peer branch that must exist in its own right.

- `N1` Compose and decompose small numbers
- `N2` Addition and subtraction meaning
- `N3` Count on and count back
- `N4` Fluency foundations
- `N5` Teen numbers and base-ten composition readiness
- `N6` Early place-value grouping logic

### C2. Place-Value And Multidigit Whole-Number Branch

This branch is now present in the graph and is required if the repository intends to represent broad elementary territory rather than only readiness-level whole-number structure.

- `V1` Two-digit number structure
- `V2` Compare and order multidigit numbers
- `V3` Add/subtract within 100 strategies
- `V4` Regrouping and base-ten equivalence
- `V5` Add/subtract within 1000

### C3. Decimal And Fraction-Decimal Branch

This branch is required because fraction-peer elementary territory is not complete if decimal place value, decimal magnitude, and fraction-decimal relationships are absent.

- `X1` Tenths as decimal units
- `X2` Decimal place-value structure
- `X3` Compare and order decimals
- `X4` Fraction-decimal benchmark connections
- `X5` Decimal locations on number line

### D. Unitizing, Sharing, And Equal-Group Branch

This branch supports fractions directly, but it also supports multiplication and division readiness later.

- `U1` Equal groups
- `U2` Repeated unit reasoning
- `U3` Sharing and fair partitioning
- `U4` Equal partitioning of a whole
- `U5` Equal-group multiplication readiness
- `U6` Equal-share division readiness

### D2. Multiplicative Reasoning And Division Branch

The current graph now includes the early multiplicative branch through the peer-level multidigit territory required for comprehensive fraction-peer coverage.

- `T1` Arrays and rectangular structures
- `T2` Multiplication fact structure
- `T3` Division as unknown group or share
- `T4` Multiplicative comparison and scaling
- `T5` Factors, multiples, and divisibility
- `T6` Multidigit multiplication structure
- `T7` Multidigit division and remainder interpretation

### E. Measurement And Magnitude Branch

- `M1` Measurable attributes
- `M2` Iterated units
- `M3` Unit size matters
- `M4` Length comparison and ordering
- `M5` Indirect comparison and transitivity
- `M6` Time, calendar, and sequence measurement readiness
- `M7` Money and value-unit readiness
- `M8` Standard length units and tool use
- `M9` Perimeter, area, and unit-square covering
- `M10` Clock time and elapsed intervals
- `M11` Money composition and exchange
- `M12` Mass and volume attributes
- `M13` Standard mass/volume units and comparison
- `M14` Unit conversion within measurement systems

### F. Geometry And Spatial Reasoning Branch

- `G1` Shape recognition and classification
- `G2` Position, direction, and spatial language
- `G3` Compose and decompose shapes
- `G4` Pattern blocks and tiling structure
- `G5` Symmetry and equal-part visual reasoning
- `G6` Polygon attributes and hierarchy
- `G7` Angle concept and turn measure

### G. Patterning And Algebraic Readiness Branch

This branch supports later structure sense and should exist in parallel with early arithmetic work.

- `P1` Repeating patterns
- `P2` Growing patterns
- `P3` Missing-element pattern reasoning
- `P4` Rule language and structure noticing

### H. Data, Classification, And Representation Branch

This branch supports mathematical communication, comparison, and early evidence reasoning.

- `D1` Sort and classify by attribute
- `D2` Count and compare category totals
- `D3` Read simple picture graphs and tallies
- `D4` Represent simple data sets
- `D5` Scaled bar graphs and numerical data
- `D6` Line plots and measurement data

### H2. Number-Line And Linear Magnitude Branch

This branch now exists in the implemented graph and ties whole-number, multidigit magnitude, and fraction magnitude into one linear representation.

- `L1` Ordered positions on number line
- `L2` Whole-number intervals and jumps
- `L3` Multidigit magnitude on number line
- `L4` Fraction locations on number line

### H3. Estimation And Reasonableness Branch

This branch is required because elementary mathematics at the fraction-peer level expects explicit estimation and reasonableness control, not only exact computation.

- `Y1` Rounding and benchmark whole numbers
- `Y2` Whole-number operation estimation
- `Y3` Measurement estimation and reasonableness
- `Y4` Fraction benchmark estimation

### I. Fraction-Readiness And Early Fraction Branch

This is the branch the repository led with and has already operationalized much more deeply than adjacent peer branches.

- `R1` Part-whole reasoning with equal parts
- `R2` Unit fraction readiness
- `R3` Symbol preparation
- `F1` Fraction structure
- `F2` Equivalent fractions
- `F3` Compare fractions
- `F4` Like-denominator fraction operations
- `F5` Unlike-denominator fraction operations
- `F6` Fraction magnitude beyond one whole

## Current Build Boundary

The original narrower prerequisite build is closed, and the intended broader elementary-math peer scope is now represented in implemented node packages.

The current repository should hold at this boundary and improve coherence before moving forward again:

1. keep the downstream boundary bounded at the fraction frontier through `F6/F5`
2. review the implemented peer-level branches for remaining weak dependencies and overcompressed abstractions after the numeral, fraction-greater-than-one, and measurement-conversion remediations
3. continue to treat `S1`, `S2`, `E1`, `E2`, and `E3` as explicitly downstream and out of current scope

Reason:

- it is clearly peer-level to the current fraction-supporting work
- it strengthens visual structure reasoning
- it prevents the graph from becoming numeracy-only

## Claim Boundary

This graph defines the implemented early-math frontier target. It is not yet a claim that all branches are equally refined, validated, or classroom-ready.
