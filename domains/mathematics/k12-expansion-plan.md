# K-12 Expansion Plan

## Purpose

Sequence the repository from the current implemented lower-to-middle mathematics graph into a full pre-K through Grade 12 and AP mathematics master graph.

## Current State

The current mathematics graph is fully implemented for the scope that has been modeled so far.

That implemented scope reaches:

- broad pre-K through Grade 5 mathematics
- a meaningful Grade 6 and early Grade 7 downstream segment
- ratios, percent, expressions, inequalities, coordinate reasoning, and early linear relationships

This is the completed base from which the full K-12/AP expansion should proceed.

## Planning Rule

The next work should expand architecture first, then node families, then node packages.

That means the order is:

1. define the next required strand families
2. define their internal dependency logic
3. connect them to the current implemented frontier
4. only then implement governed node packages

## Expansion Phases

### Phase 10. Complete The Middle-School Graph

Objective:

- extend the current early Grade 7 frontier through the rest of Grade 7 and Grade 8 mathematics

Required territory:

- full rational-number operations
- numerical expressions and order structure
- percent increase, decrease, tax, discount, and rate application
- linear equations in multiple forms
- systems foundations
- function foundations
- transformations and similarity foundations
- angle relationships and geometric problem structure
- volume and surface-area reasoning
- statistics distributions and variability
- introductory probability

Primary outcome:

- a fully modeled and then implemented middle-school bridge into true secondary mathematics

Phase-entry sequencing for this band:

1. rational-number computation and order
2. numerical expressions, absolute value, and percent application
3. variable relationships and function foundations
4. slope, linear equations, and multiple representations
5. systems, geometry transformations, and statistics/probability foundations

### Phase 11. Algebra I Architecture And Implementation

Objective:

- define and implement the first full secondary algebra branch

Required territory:

- function concept
- linear functions
- systems of equations and inequalities
- exponential-introduction links where appropriate
- descriptive modeling with graphs and equations
- quadratic foundations

Primary outcome:

- a governed Algebra I layer with explicit dependency attachment to the middle-school graph

### Phase 12. Geometry Architecture And Implementation

Objective:

- define and implement the secondary geometry and proof branch

Required territory:

- transformations
- congruence
- similarity
- proof structure
- right triangles and trigonometric ratios
- circles
- coordinate geometry
- measurement and modeling

Primary outcome:

- a governed Geometry layer connected both to elementary geometry foundations and Algebra I structures

### Phase 13. Algebra II Architecture And Implementation

Objective:

- define and implement advanced secondary algebra

Required territory:

- polynomial structure
- rational expressions
- radical expressions
- exponential and logarithmic functions
- complex numbers
- sequences and series
- advanced modeling

Primary outcome:

- a governed Algebra II layer that prepares the graph for precalculus and calculus

### Phase 14. Statistics And Probability Expansion

Objective:

- elevate statistics and probability into a full secondary branch rather than leaving them as a light side strand

Required territory:

- distributions
- center and spread
- bivariate data
- linear association
- sampling and inference foundations
- probability models
- AP Statistics preparation and extension

Primary outcome:

- a continuous data-to-statistics branch from early data representation through AP Statistics

### Phase 15. Precalculus And Trigonometry Expansion

Objective:

- define the advanced pre-calculus bridge to calculus

Required territory:

- advanced function composition and transformation
- trigonometric functions and identities
- unit circle
- radian measure
- inverse trigonometric structure where appropriate
- vectors, parametrics, conics, and sequences as governed choices
- limits foundations

Primary outcome:

- a governed precalculus layer with clear on-ramps into calculus and AP Precalculus

### Phase 16. Calculus And AP Expansion

Objective:

- complete the AP-level mathematics graph

Required territory:

- AP Precalculus overlay if retained
- AP Statistics overlay
- Calculus AB core
- Calculus BC extensions

Primary outcome:

- complete repository-wide K-12/AP mathematics scope

## Immediate Next Deliverables

The next implementation round should produce:

1. a revised mathematics domain README that states the K-12/AP target directly
2. a revised master-graph narrative that no longer describes the domain as only an early-math graph
3. new strand-family definitions for the first unmodeled secondary expansion phase
4. a phase entry plan for completing the middle-school graph before jumping deeper into high school

## Recommended Immediate Modeling Frontier

The strongest next modeling frontier is:

- complete middle-school mathematics before adding high-school/AP nodes

Why:

- it preserves dependency integrity
- it gives secondary algebra a stable upstream base
- it prevents high-school and AP families from becoming attached to an under-modeled middle layer

## Completion Metric

The domain should track completion using two different measures:

1. current modeled-graph implementation percentage
2. overall K-12/AP architecture coverage percentage

At present:

- current modeled-graph implementation is complete
- overall K-12/AP architecture coverage is not complete because the secondary and AP families are not yet modeled
