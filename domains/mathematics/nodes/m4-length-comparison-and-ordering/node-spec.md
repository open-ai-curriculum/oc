# M4 Node Specification

## Node Identity

- id: `M4`
- name: `length_comparison_and_ordering`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner compares and orders measurable quantities using direct comparison, consistent attribute focus, and correct alignment.

## Dependencies

- `M1`

## Exit Conditions

The learner is secure on `M4` only when all of the following are true:

1. identifies which of two quantities is longer, shorter, taller, or heavier when direct comparison is possible
2. orders at least three quantities correctly by a specified measurable attribute
3. explains that comparison requires attending to the same attribute and proper alignment
4. succeeds across concrete objects, drawings, and language-based prompts
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- compare two objects by placing them side by side and aligning endpoints
- order three objects from shortest to longest
- explain why misalignment distorts a comparison
- use comparative vocabulary consistently for the target attribute

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
