# H2 Node Specification

## Node Identity

- id: `H2`
- name: `congruence_and_similarity_via_transformations`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner uses rigid motions and dilations to reason about congruence and similarity, establishes correspondence among figures, and explains scale structure instead of relying on appearance or memorized shortcuts.

## Dependencies

- `H1`
- `Q3`
- `G7`


## Exit Conditions

The learner is secure on this node only when all of the following are true:

1. determines when figures are congruent or similar by using rigid motions, dilations, or valid transformation sequences
2. explains how angle preservation, side-length relationships, and scale factor support congruence or similarity claims
3. identifies corresponding sides, angles, and vertices consistently across transformed or proportional figures
4. solves bounded geometric similarity and congruence problems using transformation or proportional-structure reasoning
5. passes a delayed recheck after spacing


## Mastery Criteria

- direct accuracy threshold: `0.85`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- justify congruence through rigid-motion reasoning and similarity through dilation or proportional correspondence
- explain how scale factor changes side lengths while preserving angle structure
- identify corresponding features consistently across congruent and similar figures
- solve bounded triangle or polygon similarity/congruence tasks using structural reasoning


## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
