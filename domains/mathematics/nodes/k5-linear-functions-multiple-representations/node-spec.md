# K5 Node Specification

## Node Identity

- id: `K5`
- name: `linear_functions_multiple_representations`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner interprets a linear function coherently across equation, table, graph, verbal rule, and contextual form, preserving the same relationship instead of treating each representation as a separate procedure.

## Dependencies

- `K3`
- `K4`
- `A5`


## Exit Conditions

The learner is secure on this node only when all of the following are true:

1. identifies and represents the same linear function across equation, table, graph, verbal, and contextual forms
2. explains how slope, starting value, and covarying quantities appear in each representation
3. translates accurately between at least two unfamiliar representation pairs without losing the underlying relationship
4. distinguishes linear functions from non-linear or non-functional examples in bounded cases
5. passes a delayed recheck after spacing


## Mastery Criteria

- direct accuracy threshold: `0.87`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- match or generate equivalent linear representations across equation, table, graph, verbal rule, and context
- explain how constant rate of change and starting value appear in each representation
- determine whether a representation pair describes the same linear function
- distinguish linear-function representations from bounded non-linear counterexamples


## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
