# P3 Node Specification

## Node Identity

- id: `P3`
- name: `missing_element_pattern_reasoning`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner infers missing elements in a pattern by reasoning from the underlying rule rather than by guesswork.

## Dependencies

- `P2`

## Exit Conditions

The learner is secure on `P3` only when all of the following are true:

1. identifies missing elements in repeating patterns accurately
2. identifies missing elements in simple growing patterns accurately
3. explains how the rule determines the missing element
4. succeeds when the missing element is at the beginning, middle, or end
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- fill a missing item in a repeating pattern
- infer a missing step in a growing pattern
- explain why a candidate answer fits the full pattern and alternatives do not
- preserve the rule even when one visible step is absent

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
