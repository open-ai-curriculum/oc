# C1 Node Specification

## Node Identity

- id: `C1`
- name: `stable_counting_sequence`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner produces the count sequence in stable order and recognizes when that order is broken.

## Dependencies

- `Z2`

## Exit Conditions

The learner is secure on `C1` only when all of the following are true:

1. recites the supported count sequence in order
2. identifies missing or out-of-order count words
3. restarts and continues the sequence reliably from bounded entry points
4. succeeds across oral, visual, and simple start-stop prompts
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- say the counting words in stable order
- notice when a number is skipped or out of place
- continue from a bounded starting point
- support later one-to-one counting because the verbal sequence is stable

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
