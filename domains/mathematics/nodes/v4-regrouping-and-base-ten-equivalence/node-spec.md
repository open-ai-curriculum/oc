# V4 Node Specification

## Node Identity

- id: `V4`
- name: `regrouping_and_base_ten_equivalence`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner understands that one ten can be renamed as ten ones and uses that equivalence in multidigit operations.

## Dependencies

- `V1`
- `V3`

## Exit Conditions

The learner is secure on `V4` only when all of the following are true:

1. uses regrouping correctly in bounded multidigit contexts
2. explains why regrouping preserves total value
3. connects regrouping moves to tens-and-ones equivalence
4. succeeds across models, drawings, and written strategies
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- rename one ten as ten ones when needed
- use regrouping in addition or subtraction with understanding
- explain why the total value stays the same when units are renamed
- connect regrouping to base-ten equivalence rather than rote steps

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
