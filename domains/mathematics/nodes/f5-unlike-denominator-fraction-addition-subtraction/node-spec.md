# F5 Node Specification

## Node Identity

- id: `F5`
- name: `add_subtract_unlike_denominator_fractions`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `initial-node-package`
- evidence class: `Synthetic`

## Purpose

The learner transforms fractions to a common unit size while preserving value, then adds or subtracts accurately and explains why the transformation is valid.

## Dependencies

- `F2` Equivalent Fractions
- `F3` Compare Fractions
- `F4` Add And Subtract Like-Denominator Fractions
- `F6` Fraction Magnitude Beyond One Whole

## Entry Conditions

The learner should already be able to:

- generate equivalent fractions by scaling numerator and denominator by the same nonzero number
- compare fractions using benchmarks or common units
- add and subtract like-denominator fractions without combining denominators incorrectly

## Exit Conditions

The learner is considered secure on `F5` only when all of the following are true:

1. solves unlike-denominator fraction addition and subtraction tasks accurately
2. explains why common-denominator rewriting preserves quantity
3. succeeds on at least one transfer task with a changed surface form
4. passes a delayed retention check after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- find or use an appropriate common denominator
- rewrite one or both fractions as equivalent fractions correctly
- add or subtract the resulting like-denominator fractions
- simplify when required by the task or expected convention
- justify why the rewritten fractions retain the same value
- check the plausibility of the answer using magnitude reasoning

## Scope Boundaries

This node includes:

- proper fractions
- improper fractions
- mixed numbers in bounded instructional cases
- addition and subtraction in symbolic, visual, verbal, and contextual forms

This node does not yet include:

- algebraic rational expressions
- complex fraction operations beyond the bounded instructional profile
- proof-oriented formalism beyond age-appropriate explanation

## Claim Boundary

This node package defines a strong operational draft for the first proof-of-method node. It does not by itself establish classroom efficacy, district readiness, or production-grade learner deployment.
