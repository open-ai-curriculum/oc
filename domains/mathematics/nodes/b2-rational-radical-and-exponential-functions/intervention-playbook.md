# Intervention Playbook

## `family_constraint_fragmentation`

- mark domain restrictions or excluded values before solving and recheck them after solving
- compare matched rational, radical, and exponential tasks where the key structural constraint differs
- require learners to state the family-specific condition in words before manipulation

## `behavior_source_opacity`

- pair each advanced-function graph feature with the symbolic source that causes it
- use teacher think-alouds naming why a denominator creates an excluded value or why repeated multiplication creates exponential growth
- compare functions with similar-looking graphs but different structural causes

## `extraneous_or_excluded_solution_acceptance`

- substitute all candidate solutions back into the original structure before accepting them
- compare valid and invalid solutions that arise from the same symbolic manipulation
- use “which values are even allowed” routines before and after solving

## `representation_family_disconnect`

- compare tables, graphs, and equations of the same function family side by side and name the shared structural feature
- use graph-prediction routines before graphing technology confirms the result
- require learners to explain which part of the symbolic form accounts for the observed behavior
