# S1 Node Specification

## Node Identity

- id: `S1`
- name: `signed_number_meaning`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner interprets positive and negative numbers as directed quantities, positions, and changes rather than as whole numbers with attached symbols.

## Dependencies

- `F3`

## Exit Conditions

The learner is secure on `S1` only when all of the following are true:

1. interprets signed numbers as meaningful positions and directed quantities
2. compares and orders negative and positive values using reference-point reasoning
3. connects subtraction, change, and distance language to signed-number contexts
4. succeeds across number lines, story contexts, tables, and verbal comparisons
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- orders positive and negative values on a bounded number line
- interprets negative values in contexts such as temperature, elevation, debt, or movement
- explains zero as a reference point rather than a signless special case
- matches symbolic signed numbers to direction and change representations

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
