# Q4 Node Specification

## Node Identity

- id: `Q4`
- name: `percent_as_rate_per_hundred`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner interprets percent as a rate per hundred and uses that structure to represent, compare, and solve bounded percent situations coherently.

## Dependencies

- `Q2`
- `F2`
- `V2`


## Exit Conditions

The learner is secure on this node only when all of the following are true:

1. interprets percent as an amount out of one hundred rather than as a detached symbol procedure
2. represents common percents using fractions, decimals, visual hundred models, and bounded word situations
3. solves bounded percent-of-whole, part-percent, and comparison tasks using rate-per-hundred reasoning
4. explains the relationship among percent, equivalent fraction, and decimal forms in simple cases
5. passes a delayed recheck after spacing


## Mastery Criteria

- direct accuracy threshold: `0.88`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- explain that 25% means 25 out of 100 and connect it to 1/4 and 0.25 in bounded cases
- use hundred grids, benchmark percents, and equivalent forms to reason about common percents
- solve bounded percent-of-whole and what-percent questions without collapsing percent into unrelated whole-number operations
- compare percent situations across tables, visuals, and contextual descriptions


## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
