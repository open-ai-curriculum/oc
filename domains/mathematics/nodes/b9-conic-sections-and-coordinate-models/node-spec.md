# B9 Node Specification

## Node Identity

- id: `B9`
- name: `conic_sections_and_coordinate_models`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner interprets circles, parabolas, ellipses, and hyperbolas as coordinate-defined model families, connecting equations, parameters, and graphs so conic reasoning becomes an explicit modeling layer within advanced mathematics.

## Dependencies

- `A4`
- `H5`
- `B1`


## Exit Conditions

The learner is secure on this node only when all of the following are true:

1. distinguishes major conic families from their equations and graphs in bounded standard-form and transformed cases
2. interprets how parameters control center, vertex, axis, orientation, and scale features of conic graphs
3. solves bounded coordinate-model questions involving conic features, intersections, and representation matching
4. connects conic equations, graph features, and contextual interpretations without reducing the work to isolated formulas
5. passes a delayed recheck after spacing


## Mastery Criteria

- direct accuracy threshold: `0.83`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- classify and interpret bounded equations or graphs as circles, parabolas, ellipses, or hyperbolas
- explain how parameter changes alter conic location, orientation, and key features
- solve bounded coordinate questions involving conic features, simple intersections, or representation matching
- connect graph structure, equation structure, and modeling meaning for conic families


## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
