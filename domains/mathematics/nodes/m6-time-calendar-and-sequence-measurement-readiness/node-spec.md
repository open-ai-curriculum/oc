# M6 Node Specification

## Node Identity

- id: `M6`
- name: `time_calendar_and_sequence_measurement_readiness`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner reasons about ordered time sequences such as days, parts of the day, and simple calendar structure.

## Dependencies

- `Z3`

## Exit Conditions

The learner is secure on `M6` only when all of the following are true:

1. orders familiar time-related sequences correctly
2. identifies before, after, yesterday, today, and tomorrow in bounded contexts
3. explains simple repeated calendar or daily-cycle structure
4. succeeds across oral scenarios, picture schedules, and simple calendar layouts
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- order morning, afternoon, evening, and night
- identify days in a simple weekly sequence
- answer simple before/after questions in familiar routines
- explain repeated sequence structures in schedules or calendars

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
