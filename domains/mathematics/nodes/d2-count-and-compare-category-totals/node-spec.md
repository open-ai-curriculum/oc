# D2 Node Specification

## Node Identity

- id: `D2`
- name: `count_and_compare_category_totals`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner counts how many items are in each category and compares the resulting totals.

## Dependencies

- `D1`

## Exit Conditions

The learner is secure on `D2` only when all of the following are true:

1. counts category totals accurately
2. compares categories using more, fewer, equal, most, and least
3. explains how the counted totals support the comparison claim
4. succeeds across concrete sorts, picture sets, and simple recorded counts
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- count items in each category after sorting
- identify which category has more, fewer, most, or least
- recognize when two categories are equal
- explain comparisons using counted totals rather than visual guess alone

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
