# M7 Node Specification

## Node Identity

- id: `M7`
- name: `money_and_value_unit_readiness`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner recognizes common coins or value markers and reasons about simple compare and exchange situations involving them.

## Dependencies

- `C4`

## Exit Conditions

The learner is secure on `M7` only when all of the following are true:

1. identifies common early money or value units in a bounded context
2. compares simple value amounts in familiar forms
3. explains a basic exchange or enough/not-enough judgment
4. succeeds across physical items, pictures, and simple story situations
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- identify a penny, nickel, dime, or other locally chosen early-value unit set
- decide which of two small amounts is more or less in familiar contexts
- answer whether an amount is enough for a simple price in a bounded scenario
- explain the comparison or exchange in simple value language

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
