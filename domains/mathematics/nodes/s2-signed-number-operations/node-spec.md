# S2 Node Specification

## Node Identity

- id: `S2`
- name: `signed_number_operations`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner operates on positive and negative values using stable additive and multiplicative structure rather than isolated sign rules.

## Dependencies

- `S1`

## Exit Conditions

The learner is secure on `S2` only when all of the following are true:

1. computes signed-number operations accurately and consistently
2. explains operation outcomes using at least one structural model for each operation family
3. selects appropriate reasoning across symbolic, contextual, and diagnostic tasks
4. succeeds across addition, subtraction, multiplication, and division with signed values
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- adds and subtracts signed numbers accurately using line, inverse, or quantity models
- multiplies and divides signed numbers with structurally grounded explanations
- checks the plausibility of signed results against context or magnitude expectations
- connects signed operations to inverse reasoning needed for later equation work

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
