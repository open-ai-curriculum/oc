# G6 Node Specification

## Node Identity

- id: `G6`
- name: `polygon_attributes_and_hierarchy`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner classifies polygons by defining attributes, compares categories, and reasons about hierarchy among shape classes.

## Dependencies

- `G3`
- `V2`


## Exit Conditions

The learner is secure on `G6` only when all of the following are true:

1. identifies defining attributes of common polygons including side count, equal sides, and right angles where appropriate
2. sorts and classifies polygons into named categories using stated attribute rules
3. explains inclusive relationships such as why a square is also a rectangle and a quadrilateral
4. succeeds across drawings, sort tasks, and compare/classify prompts
5. passes a delayed recheck after spacing


## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- name and sort triangles, quadrilaterals, pentagons, hexagons, and other familiar polygons by visible attributes
- distinguish defining attributes from non-defining features such as orientation or size
- justify category membership using explicit geometric language
- reason about subclass relationships inside polygon families


## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
