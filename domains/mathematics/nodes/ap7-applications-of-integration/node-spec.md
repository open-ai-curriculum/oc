# AP7 Node Specification

## Node Identity

- id: `AP7`
- name: `applications_of_integration`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner uses integrals to solve bounded area, volume, accumulation, and motion applications, treating integral applications as a distinct reasoning surface rather than as a direct copy of integral meaning work.

## Dependencies

- `AP6`
- `AP5`


## Exit Conditions

The learner is secure on this node only when all of the following are true:

1. solves bounded accumulation, area, volume, and motion applications using appropriate integral setup and interpretation
2. distinguishes when an application requires total amount, net change, accumulated change, or geometric measurement
3. connects integral setup choices to the structure of the contextual quantity being modeled
4. justifies integration-application conclusions with correct units, bounds, and contextual meaning
5. passes a delayed recheck after spacing


## Mastery Criteria

- direct accuracy threshold: `0.8`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- solve bounded area, volume, motion, and accumulation applications using defensible integral setup
- explain why a particular integral expression matches the contextual quantity being asked for
- distinguish total versus net interpretations and use correct units and bounds
- justify integration-application conclusions in context rather than reporting only symbolic output


## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
