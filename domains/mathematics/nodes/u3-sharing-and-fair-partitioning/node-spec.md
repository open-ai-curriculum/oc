# U3 Node Specification

## Node Identity

- id: `U3`
- name: `sharing_and_fair_partitioning`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner understands that fair sharing means each recipient gets the same amount, not just the same number of objects or pieces.

## Dependencies

- `U2`

## Exit Conditions

The learner is secure on `U3` only when all of the following are true:

1. identifies fair and unfair sharing situations
2. makes or critiques shares based on equal amount
3. explains fairness in terms of same amount rather than appearance or count alone
4. succeeds across whole-sharing and set-sharing contexts
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- decide whether a sharing situation is fair
- distribute objects or portions so each recipient gets the same amount
- explain why one unfair distribution should be adjusted
- connect fairness language to equal amount before formal fraction notation

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
