# R1 Node Specification

## Node Identity

- id: `R1`
- name: `part_whole_reasoning_with_equal_parts`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner recognizes that a whole can be decomposed into equal parts and that the value of a part depends on the whole and the equality of the partition.

## Dependencies

- `N1`
- `U4`
- `M3`

## Exit Conditions

The learner is secure on `R1` only when all of the following are true:

1. distinguishes equal partitions from unequal partitions
2. identifies the whole being partitioned
3. explains that parts must be equal for later fraction reasoning to hold
4. reasons about a part as belonging to one whole rather than as an isolated object
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- identify whether a shape or set has been partitioned into equal parts
- match a shaded part to the whole it belongs to
- explain why unequal pieces cannot support valid fraction naming
- compare different wholes and avoid assuming the same-looking piece has the same value across wholes

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
