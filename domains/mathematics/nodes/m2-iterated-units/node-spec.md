# M2 Node Specification

## Node Identity

- id: `M2`
- name: `iterated_units`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner understands that measurement can be built by repeating the same unit across a length, area, or amount.

## Dependencies

- `M1`

## Exit Conditions

The learner is secure on `M2` only when all of the following are true:

1. measures or compares by repeating a consistent unit
2. explains that the unit must be repeated without gaps or overlaps
3. recognizes when inconsistent unit use makes a measurement invalid
4. succeeds across more than one measurement context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- line up equal units to measure a length
- explain why the same unit has to be used consistently
- identify gaps, overlaps, or unit switching as errors
- connect repeated-unit measurement to later unit-size reasoning

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
