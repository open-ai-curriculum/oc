# U4 Node Specification

## Node Identity

- id: `U4`
- name: `equal_partitioning_of_a_whole`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner understands that one whole can be partitioned into equal parts and that equal partitioning is a structural requirement, not a cosmetic preference.

## Dependencies

- `U3`

## Exit Conditions

The learner is secure on `U4` only when all of the following are true:

1. partitions a whole into equal parts or identifies valid equal partitions
2. explains why equal parts are required for fair or valid partitioning
3. rejects unequal partitions as invalid examples of equal-part structure
4. succeeds across shapes, sets, and sharing contexts
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- identify when a whole is split into equal versus unequal parts
- partition a simple whole into equal shares
- explain why two parts can be the same count but not the same size
- connect partitioning to fair-sharing situations

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
