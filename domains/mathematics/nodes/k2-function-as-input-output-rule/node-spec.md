# K2 Node Specification

## Node Identity

- id: `K2`
- name: `function_as_input_output_rule`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner interprets a function as a rule assigning exactly one output to each allowed input, rather than as a loose pattern or an unlabeled collection of values.

## Dependencies

- `K1`
- `A5`


## Exit Conditions

The learner is secure on this node only when all of the following are true:

1. interprets bounded input-output relationships as functions when each input has a single output
2. distinguishes function situations from non-function situations in bounded cases
3. explains how rules, tables, graphs, and mappings can represent the same function
4. succeeds across symbolic, table, verbal, graph, and simple contextual representations
5. passes a delayed recheck after spacing


## Mastery Criteria

- direct accuracy threshold: `0.88`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- identify inputs and outputs in bounded function situations
- determine whether a table, mapping, or graph does or does not represent a function
- explain the one-input-to-one-output condition in age-appropriate language
- connect verbal rules, tables, points, and simple symbolic forms as representations of the same function


## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
