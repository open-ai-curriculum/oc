# M10 Node Specification

## Node Identity

- id: `M10`
- name: `clock_time_and_elapsed_intervals`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner reads bounded clock times and reasons about elapsed intervals as measurable spans rather than isolated labels.

## Dependencies

- `M6`
- `N3`

## Exit Conditions

The learner is secure on `M10` only when all of the following are true:

1. reads and records clock time in bounded classroom contexts
2. finds simple elapsed intervals by reasoning forward or backward in time
3. connects schedules and event sequences to measured time spans
4. succeeds across analog clocks, digital clocks, timelines, and daily routines
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- reads time on analog or digital clocks in bounded early increments
- orders events by start time and duration
- calculates simple elapsed intervals using count-on reasoning
- explains the difference between a clock reading and a span of time

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
