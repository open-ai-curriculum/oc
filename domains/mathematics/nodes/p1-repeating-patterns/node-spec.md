# P1 Node Specification

## Node Identity

- id: `P1`
- name: `repeating_patterns`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner identifies the repeating unit in a pattern, extends the pattern correctly, and explains what repeats.

## Dependencies

- `Z3`

## Exit Conditions

The learner is secure on `P1` only when all of the following are true:

1. identifies the repeating unit in a simple pattern
2. extends a repeating pattern accurately
3. explains what repeats and why the next element follows
4. succeeds across objects, pictures, gestures, and simple symbols
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- identify an `AB`, `AAB`, or `ABC` repeating unit
- continue a repeating pattern correctly
- explain the cycle using language such as "it repeats every..."
- reject continuations that break the cycle

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
