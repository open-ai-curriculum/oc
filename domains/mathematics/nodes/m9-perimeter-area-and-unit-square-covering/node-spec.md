# M9 Node Specification

## Node Identity

- id: `M9`
- name: `perimeter_area_and_unit_square_covering`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner distinguishes boundary length from covered region and uses iterated units or unit squares to quantify each appropriately.

## Dependencies

- `M8`
- `G4`

## Exit Conditions

The learner is secure on `M9` only when all of the following are true:

1. distinguishes perimeter tasks from area tasks reliably
2. measures perimeter and area using appropriate unit structures
3. uses rows, columns, or decomposition to avoid recounting every square when appropriate
4. succeeds across rectangles, composite shapes, and covering tasks
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.88`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- identifies whether a task asks for boundary or covered region
- finds perimeter by iterating boundary units
- finds area by counting or structuring unit-square coverage
- explains why identical shapes can share one measure while differing on the other

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
