# M11 Node Specification

## Node Identity

- id: `M11`
- name: `money_composition_and_exchange`
- domain: `mathematics`
- support tier: `Core release`
- current maturity state: `Drafted`
- supported profile: `upstream-readiness-node-package`
- evidence class: `Synthetic`

## Purpose

The learner composes value with common units and reasons about exchange, equivalence, and enough/not-enough judgments in bounded money contexts.

## Dependencies

- `M7`
- `V1`

## Exit Conditions

The learner is secure on `M11` only when all of the following are true:

1. composes and decomposes bounded money amounts with common value units
2. reasons about exchanges and simple change within tightly constrained contexts
3. distinguishes number of pieces from total value
4. succeeds across physical money models, pictures, tables, and purchase stories
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.88`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- builds the same total value in more than one way with common coins or units
- compares two amounts using composed value rather than raw coin count
- determines whether an amount is enough for a bounded price
- explains simple exchanges that preserve or increase/decrease total value

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
