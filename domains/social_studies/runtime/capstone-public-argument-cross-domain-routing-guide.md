# Capstone Cross-Domain Routing Guide

## Purpose

This guide explains when the capstone public-argument runtime slice should remain inside the social studies domain and when it should temporarily route into `learning_to_learn` support.

## Primary Rule

Capstone synthesis and public argument remain the primary instructional surface.

Cross-domain routing is appropriate only when evidence suggests the learner's main blocker is:

- inability to hold multi-source structure across steps
- repeated reuse of an unfit routine without a check step
- failure to turn feedback into revision

## Route Patterns

### `A1 -> M1`

Use when interdisciplinary synthesis fails because the learner cannot retain or organize evidence structure across notes, drafts, and source comparisons.

### `A1 -> S1`

Use when the learner keeps applying a weak research, note sorting, or synthesis routine without checking whether it fits the problem.

### `A2 -> R1`

Use when the learner can produce a public argument draft but does not use critique, audience response, or counterevidence to revise it.

## Reverification Rule

If a learner reached provisional progress on `A1` or `A2` only after cross-domain support, both the capstone node and the learning-process support should be reverified before secure status is assigned.

## Claim Boundary

This guide is a draft runtime interpretation aid. It does not replace node-level verification rules or human review for learner-impacting decisions.
