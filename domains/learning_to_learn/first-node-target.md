# First Learning-To-Learn Node Target

## Purpose

Identify the first concrete node-package wave for the `learning_to_learn` domain.

## Recommendation

The first implementation wave should begin at the true entry point of the learning-process system:

1. `A1` shared task entry, attention anchoring, and routine persistence
2. `C1` understanding checks and clarification signaling
3. `M1` encoding through chunking, representation, and rehearsal
4. `S1` productive practice routine selection
5. `R1` self-monitoring, help-seeking, and feedback acceptance

## Why This Wave

This is the highest-value first build because:

- it starts at the earliest cross-domain access layer rather than assuming readiness to learn from instruction
- it creates explicit upstream dependencies for later retrieval, transfer, and independent learning work
- it prevents later retention and strategy claims from bypassing task-entry or confusion-signaling readiness
- it establishes the earliest governed learner-state surface for learning-process capabilities
- it provides the first honest bridge between subject-domain node packages and cross-domain learning-process support

## Immediate Follow-On Wave

The next wave should deepen the first bridge from participation into deliberate learning behavior:

1. `A2` attention planning, distraction recovery, and task completion
2. `C2` explanation monitoring and question generation
3. `M2` retrieval practice, spacing awareness, and cue use
4. `S2` strategy matching, worked-example use, and note capture
5. `R2` goal setting, progress tracking, and simple reflection

## Next Frontier

After the elementary bridge is stable, the next implementation wave should move into middle-grade diagnosis, revision, and transfer work:

1. `C3` misunderstanding diagnosis and precision questioning
2. `M3` durable retention planning and mixed retrieval
3. `S3` error analysis, strategy revision, and resource selection
4. `R3` feedback integration, frustration regulation, and persistence decisions
5. `T1` near transfer across tasks, contexts, and representations

## Secondary Frontier

The later implementation wave should deepen independent learning and far transfer:

1. `A3` independent task structuring and attention management
2. `M4` knowledge consolidation, synthesis, and long-horizon retention
3. `S4` multi-strategy planning, source evaluation, and self-directed revision
4. `R4` metacognitive evaluation and adaptive learning decisions
5. `T2` far transfer, abstraction, and independent learning orchestration

## Next Phase

Once the first wave begins, priority should shift from breadth to package completeness:

1. learning-process learner-state models
2. verification models that distinguish entry, monitoring, retention, transfer, and reflection evidence
3. failure taxonomies that separate confusion, overload, avoidance, ineffective strategy, and access constraints
4. intervention playbooks tied to diagnosed learning-process breakdowns

## Claim Boundary

This target document defines implementation priority only. It does not mean the listed learning-to-learn nodes are already implemented, classroom-ready, or fully reviewed.
