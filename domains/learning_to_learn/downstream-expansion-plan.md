# Learning-To-Learn Downstream Expansion Plan

## Purpose

Record the next operational expansion surfaces for the `learning_to_learn` domain now that the full canonical 20-node graph is implemented.

## Current Position

The domain now includes:

- full graph coverage from `A1` through `T2`
- cross-domain subject attachment records
- a first attachment registry for support assets
- a bounded upper-band runtime slice
- an operational export scaffold

## Next Expansion Targets

### 1. Subject-Domain Runtime Bridges

- extend cross-domain runtime examples beyond the mathematics fraction slice
- add ELA/literacy and science routing slices with worked learner-state cases
- preserve explicit claim boundaries so process routing never replaces content mastery evidence

### 2. Asset Attachment Coverage

- expand attachment registries from upper-band support assets to earlier bands
- add accessibility and language-support attachment records
- require human review before any learner-facing asset is treated as approved

### 3. Operational Export Growth

- enrich the operational export with subject attachment entities
- add stronger summary counts and consistency checks
- keep exports derivative from canonical graph and registry sources

### 4. Human Review Surfaces

- define review checkpoints for learning-process claims that could affect learner placement
- add review-state conventions for support assets and runtime routing examples
- preserve explicit uncertainty where cognitive-science interpretation remains draft
