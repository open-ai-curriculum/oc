# S3 Node Specification

## Node Identity

- id: `S3`
- name: `error_analysis_strategy_revision_and_resource_selection`
- domain: `learning_to_learn`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `cross-domain-learning-support-node-package`
- evidence class: `Synthetic`

## Purpose

The learner analyzes why an attempt failed, changes strategy, and chooses an appropriate resource rather than repeating the same ineffective move.

## Dependencies

- `S2` strategy matching, worked-example use, and note capture
- `R2` goal setting, progress tracking, and simple reflection
- `C3` misunderstanding diagnosis and precision questioning

## Exit Conditions

The learner is secure on `S3` only when all of the following are true:

1. identifies the likely reason an attempt failed
2. selects a new strategy or revision move instead of repeating the same attempt
3. chooses a resource matched to the diagnosed breakdown
4. succeeds across more than one task type
5. remains stable on delayed recheck

## Mastery Criteria

- direct accuracy threshold: `0.80`
- explanation required: `false`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- name what went wrong in the first attempt
- pick a different strategy or support
- use that change to improve the next attempt

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
