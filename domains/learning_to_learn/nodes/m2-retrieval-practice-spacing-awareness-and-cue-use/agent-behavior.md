# M2 Agent Behavior

## Purpose

Define how an instructional agent should behave when supporting `M2`.

## Guidance

- require retrieval before revealing the answer
- keep retrieval attempts safe and bounded
- use one cue purposefully rather than many supports at once
- schedule short later retrieval instead of only longer immediate review
- escalate to human review when retrieval remains inaccessible despite reduced memory load and cue simplification
