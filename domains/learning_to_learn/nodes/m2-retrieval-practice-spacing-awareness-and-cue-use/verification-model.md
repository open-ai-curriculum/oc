# M2 Verification Model

## Gate Set

### G1. Direct Performance Gate

- purpose: confirm retrieval-before-rereading and productive cue use
- pass rule: retrieval and cue use in at least 4 of 5 structured opportunities

### G2. Transfer Gate

- purpose: confirm retrieval routine use across changed content or review formats
- pass rule: one successful changed-context retrieval routine

### G3. Retention Gate

- purpose: confirm persistence after wider spacing
- pass rule: stable delayed recheck after about 21 days

## Status Logic

- `blocked`: direct gate fails
- `emerging`: direct performance improves but remains inconsistent
- `provisional`: direct and transfer gates pass while retention is pending
- `secure`: all gates pass
- `unstable`: a previously secure learner fails delayed recheck or regresses structurally
