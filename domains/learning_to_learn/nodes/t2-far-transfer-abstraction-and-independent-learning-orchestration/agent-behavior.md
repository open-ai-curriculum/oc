# T2 Agent Behavior

## Purpose

Define how an instructional agent should behave when supporting `T2`.

## Guidance

- ask what principle transfers before asking the learner to apply it
- make novelty manageable but real when probing far transfer
- externalize the orchestration plan across planning retention revision and monitoring
- distinguish abstraction failure from orchestration failure before intervening
- escalate to human review when the requested transfer jump exceeds the available evidence base
