# S1 Node Specification

## Node Identity

- id: `S1`
- name: `productive_practice_routine_selection`
- domain: `learning_to_learn`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `cross-domain-learning-support-node-package`
- evidence class: `Synthetic`

## Purpose

The learner uses a bounded productive-practice routine such as repeat-check-correct, say-show-do, or example-then-try and can continue after an initial error instead of abandoning the task.

## Dependencies

- `A1` shared task entry, attention anchoring, and routine persistence
- `C1` understanding checks and clarification signaling

## Exit Conditions

The learner is secure on `S1` only when all of the following are true:

1. uses a recognizable productive-practice routine during supported work
2. checks or compares an attempt against a model, cue, or expected result
3. makes a correction after an initial error
4. succeeds across more than one task type
5. remains stable on delayed recheck

## Mastery Criteria

- direct accuracy threshold: `0.80`
- explanation required: `false`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- follow a bounded practice routine instead of guessing repeatedly
- compare an attempt to an example cue or expected outcome
- revise the attempt after feedback or self-check

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
