# R3 Node Specification

## Node Identity

- id: `R3`
- name: `feedback_integration_frustration_regulation_and_persistence_decisions`
- domain: `learning_to_learn`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `cross-domain-learning-support-node-package`
- evidence class: `Synthetic`

## Purpose

The learner incorporates critique into a revised attempt and manages frustration well enough to choose whether to persist, pause, or seek another support move.

## Dependencies

- `R2` goal setting, progress tracking, and simple reflection
- `S2` strategy matching, worked-example use, and note capture

## Exit Conditions

The learner is secure on `R3` only when all of the following are true:

1. receives feedback without immediate task collapse
2. integrates at least one bounded piece of feedback into a revised attempt
3. uses a frustration-regulation routine during a challenging task
4. makes a reasonable persistence decision rather than only stopping or only pushing blindly
5. remains stable on delayed recheck

## Mastery Criteria

- direct accuracy threshold: `0.80`
- explanation required: `false`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- stay available after critique
- revise one part of the work using feedback
- regulate enough to choose the next support move

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
