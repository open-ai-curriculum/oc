# R1 Agent Behavior

## Purpose

Define how an instructional agent should behave when supporting `R1`.

## Guidance

- name the stuck-check-help-revise cycle explicitly
- make help requests concrete and immediately actionable
- keep feedback bounded enough for one revision attempt
- do not treat silence as evidence of coping or understanding
- escalate to human review when help-seeking or feedback acceptance remains inaccessible after repeated bounded support
