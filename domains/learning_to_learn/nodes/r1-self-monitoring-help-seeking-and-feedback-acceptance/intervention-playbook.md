# R1 Intervention Playbook

## Purpose

Map `R1` failure modes to bounded intervention responses.

## Intervention Rules

### For R1-FM-01 Stuck State Not Signaled

Primary interventions:

- define a clear stuck signal before the task begins
- insert brief self-check pauses during work
- teach the difference between keep trying and ask for help points

Expected effect:

- learner signals being stuck before extended unproductive time

Reverification target:

- one timed work segment with built-in self-check

### For R1-FM-02 Help Request Not Usable

Primary interventions:

- provide sentence frames or gesture options for asking for help
- prompt the learner to name the step example or tool needed
- keep help requests tied to one immediate next action

Expected effect:

- learner requests help in a way that yields a usable next step

Reverification target:

- one stuck-signal and help-request exchange

### For R1-FM-03 Feedback Rejection Or Withdrawal

Primary interventions:

- deliver one bounded correction target at a time
- require a brief revision attempt immediately after feedback
- reinforce recovery and revision rather than only accuracy

Expected effect:

- learner remains available for revision after feedback

Reverification target:

- one feedback-then-revise cycle

## Escalation Rule

If the same dominant failure mode remains blocking after two bounded intervention cycles, escalate to human review of affect regulation, trust, communication access, and context factors rather than only increasing corrective feedback.
