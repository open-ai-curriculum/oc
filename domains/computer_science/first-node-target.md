# First Computer Science Node Target

## Purpose

Identify the first concrete node-package wave for the `computer_science` domain.

## Recommendation

The first implementation wave should begin at the true entry point of the computer science system:

1. `D1` digital tool navigation and interface cause-effect
2. `P1` sequencing commands and programmable action
3. `R1` symbols, encoding, and representation awareness
4. `I1` responsible participation in digital spaces
5. `C1` noticing errors, revising steps, and persistence in computing tasks

## Why This Wave

This is the highest-value first build because:

- it starts at the earliest K-2 computer science access layer rather than assuming reading-device or code-entry readiness
- it creates explicit upstream dependencies for later programming, data, systems, and debugging work
- it prevents later software and AP-facing claims from bypassing representation, revision, and responsible-use readiness
- it establishes the earliest governed learner-state surface for computer science
- it provides the first honest bridge between repository-native capability modeling and future standards-support work

## Immediate Follow-On Wave

The next wave should deepen the first bridge from simple executable action into pattern, data, and connected-system awareness:

1. `D2` file, device, and input-output workflow awareness
2. `P2` loops, events, and reusable program patterns
3. `A1` pattern generalization and stepwise problem breakdown
4. `R2` data collection, classification, and simple visualization
5. `N1` connected devices and basic network awareness
6. `I2` digital communication, attribution, and audience awareness

## Next Frontier

After the elementary bridge is stable, the next implementation wave should move into middle-grade abstraction, debugging, and systems work:

1. `P3` variables, conditionals, and stateful program behavior
2. `A2` algorithm design, efficiency awareness, and decomposition discipline
3. `C2` debugging strategy, trace reasoning, and test cases
4. `R3` data transformation, tables, and computational models
5. `N2` network pathways, protocol ideas, and system dependence
6. `N3` security risk, access control, and privacy foundations
7. `I3` computing impacts, bias, and human-centered evaluation

## Secondary Frontier

The later implementation wave should deepen advanced secondary software, data, systems, and AP-facing integration:

1. `P4` functions, modular program design, and abstraction boundaries
2. `P5` structured data, collections, and programmed data operations
3. `P6` object-oriented and multi-component program reasoning
4. `A3` algorithm analysis, tradeoffs, and formalized problem solving
5. `R4` data pipelines, modeling, and computational inference
6. `N4` systems architecture, resource constraints, and platform layers
7. `N5` cybersecurity mechanisms, threat models, and defensive reasoning
8. `C3` specification, testing, and program quality criteria
9. `C4` refactoring, maintainability, and collaborative code change
10. `I4` ethics, governance, and societal consequences of computing systems
11. `X1` AP CSP integrated computing systems, data, and impacts synthesis
12. `X2` AP CSA program design, objects, and algorithmic implementation
13. `X3` advanced computing projects, capstone design, and cross-domain application

## Next Phase

Once the first wave begins, priority should shift from breadth to package completeness:

1. computer-science-specific learner-state models
2. verification models that distinguish programming, explanation, transfer, and retention
3. failure taxonomies that separate interface error, representation confusion, algorithmic breakdown, debugging weakness, and systems misunderstanding
4. intervention playbooks tied to diagnosed computing reasoning breakdowns

## Claim Boundary

This target document defines implementation priority only. It does not mean the listed computer science nodes are already implemented, classroom-ready, or fully reviewed.
