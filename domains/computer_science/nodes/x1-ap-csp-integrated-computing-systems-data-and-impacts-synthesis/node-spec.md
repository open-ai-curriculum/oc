## Purpose

The learner synthesizes programming, data, systems, and impact reasoning in an AP Computer Science Principles style without treating the course as disconnected topic coverage.

## Exit Conditions

The learner is secure on `X1` only when all of the following are true:

1. integrates bounded programming, data, system, and impact claims in a coherent computing explanation
2. explains how technical and societal dimensions interact in a bounded AP CSP style scenario
3. uses computational artifact or process reasoning with explicit abstraction and data-system awareness
4. succeeds across more than one AP CSP style synthesis context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `42`

## Observable Evidence

The learner can:

- connect program behavior, data processing, and system context in one bounded explanation
- interpret computing impacts with reference to technical mechanisms rather than broad claims alone
- reason across abstractions instead of treating data, code, and social effects as separate silos
- avoid AP-style topic recitation without integrated causal structure

This node package is an operational draft and not a classroom-release claim.
