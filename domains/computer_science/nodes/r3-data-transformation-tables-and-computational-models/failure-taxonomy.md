# R3 Failure Taxonomy

- `R3-FM-01` output-without-rule meaning: learner reads transformed output without understanding the rule that produced it.
- `R3-FM-02` table-column drift: learner loses track of what each column or transformed field represents.
- `R3-FM-03` model-as-fact: learner treats a simple computational model as raw reality instead of a rule-based representation.
- `R3-FM-04` transformation-transfer fragility: learner understands one familiar transformation only and cannot adapt to a nearby case.
