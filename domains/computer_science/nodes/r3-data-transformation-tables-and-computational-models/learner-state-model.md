# R3 Learner State Model

The learner-state for `R3` tracks:

- dependency security on `R2`, `A2`
- direct accuracy on data transformation, tables, and computational models tasks
- explanation quality for bounded computational data reasoning reasoning
- transfer performance across unfamiliar computing contexts
- delayed retention outcome

`secure` requires all verification gates to pass. Any delayed failure or dependency instability should move the learner to `unstable`.
