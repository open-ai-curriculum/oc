# C4 Intervention Playbook

- For `C4-FM-01`, use before-and-after refactor analyses requiring one named maintainability improvement and one preserved behavior claim.
- For `C4-FM-02`, use behavior-preservation checks that pair a revision with bounded tests or specification references.
- For `C4-FM-03`, use code-review style prompts that require a change rationale suitable for collaborators.
- For `C4-FM-04`, use paired refactor tasks that reapply one maintainability pattern in a second bounded code example.
