# C4 Failure Taxonomy

- `C4-FM-01` cleanup-without-proof: learner assumes a refactor is safe without checking behavior against tests or specifications.
- `C4-FM-02` maintainability-without-criteria: learner says code is cleaner without naming what maintainability issue improved.
- `C4-FM-03` collaboration-discipline gap: learner treats code changes as personal edits rather than shared work requiring reviewable reasoning.
- `C4-FM-04` refactor-transfer fragility: learner can justify one familiar cleanup only and cannot adapt the same reasoning to another bounded codebase change.
