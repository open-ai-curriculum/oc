# I2 Node Specification

## Node Identity

- id: `I2`
- name: `digital_communication_attribution_and_audience_awareness`
- domain: `computer_science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `computer-science-elementary-node-package`
- evidence class: `Synthetic`

## Purpose

The learner communicates digitally with attention to audience, attribution, and respectful participation without treating digital communication as content with no source or audience consequences.

## Dependencies

- `I1`
- `D2`

## Exit Conditions

The learner is secure on `I2` only when all of the following are true:

1. makes a bounded digital communication choice that fits the audience and context
2. recognizes when information or media should be attributed to a source
3. explains why digital communication choices can affect how others understand a message
4. succeeds across more than one bounded digital communication context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- identify who a digital message is for in a simple scenario
- distinguish own work from content that comes from another source
- make a respectful communication choice in a bounded digital setting
- avoid ignoring audience or source when sharing digital content

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
