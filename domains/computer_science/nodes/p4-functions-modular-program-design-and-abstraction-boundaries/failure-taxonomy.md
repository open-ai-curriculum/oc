# P4 Failure Taxonomy

- `P4-FM-01` copy-paste-procedure thinking: learner treats functions as copied code blocks rather than as reusable procedures with explicit interfaces.
- `P4-FM-02` interface-implementation collapse: learner cannot distinguish what a function promises from how it is internally implemented.
- `P4-FM-03` parameter-flow confusion: learner loses track of how inputs and returned values move across program parts.
- `P4-FM-04` modular-transfer fragility: learner can follow one familiar decomposition only and cannot adapt the same abstraction logic in a second bounded program.
