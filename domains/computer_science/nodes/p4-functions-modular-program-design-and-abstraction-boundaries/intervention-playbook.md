# P4 Intervention Playbook

- For `P4-FM-01`, use interface-first design tasks that require naming each function's responsibility, inputs, and outputs before implementation.
- For `P4-FM-02`, use call-and-return trace routines that follow values through bounded function interactions.
- For `P4-FM-03`, use black-box versus white-box comparisons that separate interface reasoning from internal detail.
- For `P4-FM-04`, use paired redesign tasks that reapply one decomposition pattern in a second program context.
