# P4 Node Specification

## Node Identity

- id: `P4`
- name: `functions_modular_program_design_and_abstraction_boundaries`
- domain: `computer_science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `computer-science-advanced-secondary-node-package`
- evidence class: `Synthetic`

## Purpose

The learner designs and reasons about programs using functions, parameters, and modular boundaries without treating reusable procedures as copied chunks whose interfaces can remain implicit.

## Dependencies

- `P3`
- `A2`

## Exit Conditions

The learner is secure on `P4` only when all of the following are true:

1. defines bounded functions with explicit purpose, inputs, and outputs
2. explains how modular decomposition supports readability, testing, and reuse
3. reasons about function calls and abstraction boundaries without tracing every internal step when that detail is intentionally hidden
4. succeeds across more than one modular programming context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `35`

## Observable Evidence

The learner can:

- separate a larger program into bounded functions with coherent responsibilities
- use parameters and return values to communicate across parts of a program
- explain what a function guarantees without collapsing interface and implementation
- avoid creating modules whose boundaries are arbitrary or unstable

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
