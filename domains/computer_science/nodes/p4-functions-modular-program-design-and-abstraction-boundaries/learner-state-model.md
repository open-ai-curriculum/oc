# P4 Learner State Model

The learner-state for `P4` tracks:

- dependency security on `P3`, `A2`
- direct accuracy on functions, modular program design, and abstraction boundaries tasks
- explanation quality for bounded modular program design reasoning reasoning
- transfer performance across unfamiliar computing contexts
- delayed retention outcome

`secure` requires all verification gates to pass. Any delayed failure or dependency instability should move the learner to `unstable`.
