# P5 Node Specification

## Node Identity

- id: `P5`
- name: `structured_data_collections_and_programmed_data_operations`
- domain: `computer_science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `computer-science-advanced-secondary-node-package`
- evidence class: `Synthetic`

## Purpose

The learner uses arrays, lists, records, or similarly structured collections to organize and manipulate data in programs without treating collections as shapeless storage or operations as disconnected commands.

## Dependencies

- `P4`
- `R3`

## Exit Conditions

The learner is secure on `P5` only when all of the following are true:

1. selects a bounded collection structure that matches a data-handling task
2. iterates over or transforms structured data with explicit reasoning about what each element represents
3. explains how collection operations change program state and output
4. succeeds across more than one programmed data-operation context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `35`

## Observable Evidence

The learner can:

- store related values in a bounded structured collection rather than separate ad hoc variables
- trace how iteration or data operations affect each element or record
- connect a data structure choice to the shape of the task
- avoid losing meaning about what stored fields or collection positions represent

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
