# P5 Failure Taxonomy

- `P5-FM-01` collection-as-bag thinking: learner treats a collection as undifferentiated storage and does not track element meaning or structure.
- `P5-FM-02` iteration-state drift: learner loses track of how a loop or data operation changes the collection over time.
- `P5-FM-03` structure-task mismatch: learner chooses or uses a data structure without a bounded reason tied to the task.
- `P5-FM-04` data-operation transfer weakness: learner can perform one familiar collection pattern only and cannot adapt it to a nearby data context.
