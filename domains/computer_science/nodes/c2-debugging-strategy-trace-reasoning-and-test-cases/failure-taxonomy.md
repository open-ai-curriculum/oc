# C2 Failure Taxonomy

- `C2-FM-01` guess-and-change debugging: learner changes code repeatedly without tracing or a bounded hypothesis.
- `C2-FM-02` test-case-without-purpose: learner uses examples without explaining what behavior they are checking.
- `C2-FM-03` trace-breakdown: learner loses track of values or control flow while following the program.
- `C2-FM-04` fix-without-verification: learner revises code but does not check whether the target bug was actually resolved.
