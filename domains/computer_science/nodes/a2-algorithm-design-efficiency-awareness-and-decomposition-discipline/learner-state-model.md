# A2 Learner State Model

The learner-state for `A2` tracks:

- dependency security on `A1`, `P2`
- direct accuracy on algorithm design, efficiency awareness, and decomposition discipline tasks
- explanation quality for bounded algorithm design reasoning reasoning
- transfer performance across unfamiliar computing contexts
- delayed retention outcome

`secure` requires all verification gates to pass. Any delayed failure or dependency instability should move the learner to `unstable`.
