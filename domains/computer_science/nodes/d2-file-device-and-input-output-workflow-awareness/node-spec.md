# D2 Node Specification

## Node Identity

- id: `D2`
- name: `file_device_and_input_output_workflow_awareness`
- domain: `computer_science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `computer-science-elementary-node-package`
- evidence class: `Synthetic`

## Purpose

The learner manages simple digital workflows and explains how inputs and outputs move through tools or devices without treating files and devices as unrelated isolated objects.

## Dependencies

- `D1`

## Exit Conditions

The learner is secure on `D2` only when all of the following are true:

1. uses a simple file or device workflow intentionally in a bounded task
2. explains how an input leads to an output through a tool or device
3. tracks a simple save open send or display workflow without losing the action chain
4. succeeds across more than one familiar digital workflow context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- identify where information comes in and where it goes out in a simple workflow
- describe a bounded file or device process as connected steps
- distinguish storing showing and sending as different workflow actions
- avoid random navigation when moving through a simple digital task

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
