# C3 Failure Taxonomy

- `C3-FM-01` runs-once-equals-correct belief: learner treats one successful execution as sufficient evidence of software quality or correctness.
- `C3-FM-02` test-without-spec basis: learner designs or interprets tests without anchoring them to a bounded specification.
- `C3-FM-03` quality-criterion absence: learner judges code quality without naming what criterion is being applied.
- `C3-FM-04` spec-quality transfer weakness: learner can reason about one familiar program only and cannot apply the same specification-and-testing structure elsewhere.
