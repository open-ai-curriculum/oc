# C3 Learner State Model

The learner-state for `C3` tracks:

- dependency security on `C2`, `P4`
- direct accuracy on specification, testing, and program quality criteria tasks
- explanation quality for bounded software quality reasoning reasoning
- transfer performance across unfamiliar computing contexts
- delayed retention outcome

`secure` requires all verification gates to pass. Any delayed failure or dependency instability should move the learner to `unstable`.
