# C3 Intervention Playbook

- For `C3-FM-01`, use specification-first review routines that restate required behavior before code evaluation begins.
- For `C3-FM-02`, use test-design tasks that require each case to be justified against one normal, edge, or failure condition.
- For `C3-FM-03`, use quality-criterion comparison prompts using correctness, readability, robustness, and maintainability labels explicitly.
- For `C3-FM-04`, use paired program reviews applying the same specification-and-test reasoning in a second bounded code context.
