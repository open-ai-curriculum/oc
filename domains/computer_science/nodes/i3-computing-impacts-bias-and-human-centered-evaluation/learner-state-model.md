# I3 Learner State Model

The learner-state for `I3` tracks:

- dependency security on `I2`, `R3`
- direct accuracy on computing impacts, bias, and human-centered evaluation tasks
- explanation quality for bounded computing impacts reasoning reasoning
- transfer performance across unfamiliar computing contexts
- delayed retention outcome

`secure` requires all verification gates to pass. Any delayed failure or dependency instability should move the learner to `unstable`.
