# I3 Node Specification

## Node Identity

- id: `I3`
- name: `computing_impacts_bias_and_human_centered_evaluation`
- domain: `computer_science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `computer-science-middle-grade-node-package`
- evidence class: `Synthetic`

## Purpose

The learner critiques computing systems in terms of fairness, impact, and human use contexts without treating technology as automatically neutral or automatically harmful.

## Dependencies

- `I2`
- `R3`

## Exit Conditions

The learner is secure on `I3` only when all of the following are true:

1. identifies a bounded impact of a computing system on people or groups
2. recognizes that data, design, or context choices can affect fairness
3. uses a simple human-centered criterion to evaluate a computing decision or tool
4. succeeds across more than one bounded computing-impact context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `28`

## Observable Evidence

The learner can:

- describe at least one way a computing system can affect different people differently
- connect a design or data choice to a potential fairness issue
- evaluate a tool or decision using a named human-centered criterion
- avoid reducing computing impacts to praise or blame without bounded analysis

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
