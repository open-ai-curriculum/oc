# N5 Learner State Model

The learner-state for `N5` tracks:

- dependency security on `N4`, `C3`
- direct accuracy on cybersecurity mechanisms, threat models, and defensive reasoning tasks
- explanation quality for bounded cybersecurity reasoning reasoning
- transfer performance across unfamiliar computing contexts
- delayed retention outcome

`secure` requires all verification gates to pass. Any delayed failure or dependency instability should move the learner to `unstable`.
