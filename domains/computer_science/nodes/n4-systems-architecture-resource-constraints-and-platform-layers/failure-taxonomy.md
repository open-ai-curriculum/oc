# N4 Failure Taxonomy

- `N4-FM-01` single-layer machine model: learner treats the computing system as one undifferentiated box with no meaningful internal layers or services.
- `N4-FM-02` resource-constraint blindness: learner ignores how time, memory, storage, or bandwidth limits affect system behavior.
- `N4-FM-03` platform-application collapse: learner confuses application logic with underlying platform or operating-system responsibilities.
- `N4-FM-04` architecture-transfer weakness: learner can explain one familiar stack only and cannot adapt the same layered reasoning to another bounded system.
