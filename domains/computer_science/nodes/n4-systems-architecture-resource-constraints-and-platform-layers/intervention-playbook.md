# N4 Intervention Playbook

- For `N4-FM-01`, use layered architecture diagrams that require each bounded component or service to be named and related.
- For `N4-FM-02`, use resource-budget tasks comparing how one design changes under tighter time, memory, or bandwidth limits.
- For `N4-FM-03`, use application-versus-platform scenario sorts with explicit responsibility explanations.
- For `N4-FM-04`, use paired architecture analyses transferring the same layer-and-resource reasoning to a second system example.
