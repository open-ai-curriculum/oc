# C1 Node Specification

## Node Identity

- id: `C1`
- name: `noticing_errors_revising_steps_and_persistence_in_computing_tasks`
- domain: `computer_science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `computer-science-entry-node-package`
- evidence class: `Synthetic`

## Purpose

The learner notices when a sequence or digital action does not work as intended and attempts a bounded revision rather than abandoning the task or randomizing changes.

## Dependencies

- `P1`

## Exit Conditions

The learner is secure on `C1` only when all of the following are true:

1. notices when a computing task does not match the intended outcome
2. identifies at least one likely source of the error in a bounded case
3. revises a step intentionally and checks whether the change improved the outcome
4. persists through more than one simple error-revision cycle
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- compare intended and actual outcomes in a simple computing task
- name one step or action that may have caused the error
- revise one part of a sequence instead of changing everything at once
- check whether a revision improved the result

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
