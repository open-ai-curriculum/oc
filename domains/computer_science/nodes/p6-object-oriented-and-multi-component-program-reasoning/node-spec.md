# P6 Node Specification

## Node Identity

- id: `P6`
- name: `object_oriented_and_multi_component_program_reasoning`
- domain: `computer_science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `computer-science-advanced-secondary-node-package`
- evidence class: `Synthetic`

## Purpose

The learner reasons about interacting classes, objects, and program components without reducing object-oriented programs to syntax memorization or isolated method calls.

## Dependencies

- `P5`
- `C3`

## Exit Conditions

The learner is secure on `P6` only when all of the following are true:

1. explains the distinct role of classes, objects, state, and behavior in a bounded program
2. traces how more than one program component interacts through method calls or shared interfaces
3. uses object-oriented or component-based structure to organize a bounded solution intentionally
4. succeeds across more than one multi-component programming context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `42`

## Observable Evidence

The learner can:

- distinguish a class definition from an object instance and its changing state
- explain how one component invokes or depends on another in a bounded design
- connect method behavior to object state rather than treating methods as detached commands
- avoid flattening every object-oriented system into one long procedural trace

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
