# P6 Intervention Playbook

- For `P6-FM-01`, use class-versus-instance modeling tasks that compare definitions, objects, and per-object state explicitly.
- For `P6-FM-02`, use object-state trace tables that connect method calls to state changes across one or more components.
- For `P6-FM-03`, use component interaction diagrams showing which object or module is responsible for each bounded behavior.
- For `P6-FM-04`, use paired redesign tasks that transfer one class-and-object pattern into a second application context.
