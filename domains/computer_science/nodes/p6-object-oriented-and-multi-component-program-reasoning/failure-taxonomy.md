# P6 Failure Taxonomy

- `P6-FM-01` class-object collapse: learner confuses class definitions, object instances, and the state carried by a particular object.
- `P6-FM-02` method-without-state reasoning: learner treats methods as detached actions without connecting them to object state or responsibilities.
- `P6-FM-03` component-interaction opacity: learner cannot explain how bounded program components communicate or depend on one another.
- `P6-FM-04` oop-transfer fragility: learner can imitate one familiar class pattern only and cannot adapt the same structure in a second bounded design.
