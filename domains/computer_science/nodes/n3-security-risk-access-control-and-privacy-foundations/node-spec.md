# N3 Node Specification

## Node Identity

- id: `N3`
- name: `security_risk_access_control_and_privacy_foundations`
- domain: `computer_science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `computer-science-middle-grade-node-package`
- evidence class: `Synthetic`

## Purpose

The learner reasons about basic security risks, privacy, and protective computing practices without reducing security to rule recitation only.

## Dependencies

- `N2`
- `I2`

## Exit Conditions

The learner is secure on `N3` only when all of the following are true:

1. identifies a bounded security or privacy risk in a simple digital scenario
2. explains how an access-control or protective practice addresses that risk
3. distinguishes sharing, access, and privacy decisions in bounded cases
4. succeeds across more than one bounded security or privacy context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `28`

## Observable Evidence

The learner can:

- identify who should and should not have access in a simple scenario
- connect a protective practice to the risk it is meant to reduce
- use privacy reasoning that goes beyond rule repetition alone
- avoid assuming that being online automatically means information should be public

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
