# P2 Node Specification

## Node Identity

- id: `P2`
- name: `loops_events_and_reusable_program_patterns`
- domain: `computer_science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `computer-science-elementary-node-package`
- evidence class: `Synthetic`

## Purpose

The learner uses repeated actions, events, and simple reusable structures in beginner programs without rewriting every action as an unrelated one-off step.

## Dependencies

- `P1`

## Exit Conditions

The learner is secure on `P2` only when all of the following are true:

1. uses loops or repeated structures in a bounded beginner program
2. responds to a simple event with an appropriate programmed action
3. recognizes when a reusable pattern is better than repeating many separate commands
4. succeeds across more than one beginner programming context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- identify repetition in a programming task and represent it with a loop or repeated pattern
- connect an event trigger to the resulting action in a simple program
- compare repeated code with a more reusable bounded pattern
- avoid rewriting identical steps when a repeated structure fits

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
