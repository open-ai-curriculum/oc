# R4 Failure Taxonomy

- `R4-FM-01` pipeline-opacity: learner cannot account for how data moved or changed across a bounded pipeline.
- `R4-FM-02` output-as-truth belief: learner treats computed output or modeled inference as authoritative without checking assumptions or transformations.
- `R4-FM-03` stage-disconnection: learner reasons about data cleaning, transformation, or modeling as isolated steps rather than as linked stages.
- `R4-FM-04` inference-transfer fragility: learner can explain one familiar pipeline only and cannot adapt the same reasoning to a nearby data workflow.
