# P1 Node Specification

## Node Identity

- id: `P1`
- name: `sequencing_commands_and_programmable_action`
- domain: `computer_science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `computer-science-entry-node-package`
- evidence class: `Synthetic`

## Purpose

The learner follows and creates simple ordered command sequences for a programmable agent or environment without treating instructions as an unordered list.

## Dependencies

- `D1`

## Exit Conditions

The learner is secure on `P1` only when all of the following are true:

1. follows a bounded sequence of computing commands correctly
2. creates or revises a simple ordered sequence to achieve a target outcome
3. explains why order matters in a programmable task
4. succeeds across more than one unplugged or digital programming context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `14`

## Observable Evidence

The learner can:

- execute simple command sequences in order
- identify when a target fails because the order is wrong
- describe a program as a set of steps rather than a wish or goal
- revise one step in a sequence to improve the outcome

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
