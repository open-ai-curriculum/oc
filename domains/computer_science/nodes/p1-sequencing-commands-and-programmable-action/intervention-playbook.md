# P1 Intervention Playbook

- For `P1-FM-01`, use sequence enactment routines that compare correct and incorrect command orders.
- For `P1-FM-02`, use goal-versus-step prompts that force executable action language.
- For `P1-FM-03`, use step-tracing supports that highlight the current command and next command explicitly.
- For `P1-FM-04`, use bug-and-revision talk that requires naming the broken step before changing it.
