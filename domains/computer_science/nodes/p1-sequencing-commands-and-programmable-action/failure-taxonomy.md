# P1 Failure Taxonomy

- `P1-FM-01` order-does-not-matter belief: learner treats commands as if their order can be changed freely without changing the outcome.
- `P1-FM-02` goal-step collapse: learner states the intended result but not the executable steps needed to get there.
- `P1-FM-03` sequence-tracking instability: learner loses track of where they are in a simple program or command list.
- `P1-FM-04` revision-without-cause: learner changes steps but cannot explain what problem the revision addresses.
