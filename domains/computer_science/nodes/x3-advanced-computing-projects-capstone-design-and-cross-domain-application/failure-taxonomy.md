# X3 Failure Taxonomy

- `X3-FM-01` feature-count-as-capstone: learner treats a large set of features as evidence of project quality without bounded design or verification reasoning.
- `X3-FM-02` cross-domain-without-cs rigor: learner discusses an applied project context without maintaining explicit computer science design, data, system, or quality reasoning.
- `X3-FM-03` project-constraint opacity: learner cannot explain the bounded requirements, constraints, or stakeholder conditions shaping the project.
- `X3-FM-04` capstone-transfer weakness: learner can reason through one familiar project only and cannot adapt the same synthesis structure to another bounded advanced context.
