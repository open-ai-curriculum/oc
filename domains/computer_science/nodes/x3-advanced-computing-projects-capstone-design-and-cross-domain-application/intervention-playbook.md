# X3 Intervention Playbook

- For `X3-FM-01`, use project-framing templates requiring requirements, constraints, users, and success criteria to be named explicitly.
- For `X3-FM-02`, use design-review routines that force architecture, quality, security, and ethical considerations into one bounded project account.
- For `X3-FM-03`, use cross-domain application prompts asking what remains distinctly computer-science about an applied project.
- For `X3-FM-04`, use paired capstone analyses transferring the same synthesis structure to a second advanced project scenario.
