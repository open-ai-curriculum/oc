# Upstream Build Plan

## Purpose

Document the upstream modeling work that should continue strengthening the canonical computer science graph itself.

## Upstream Targets

- reviewed node-granularity decisions for the first computer science implementation wave
- computer-science-specific verification-gate patterns that preserve direct performance, explanation, transfer, and retention
- computer-science-specific failure-mode taxonomies that separate interface error, algorithm confusion, representation failure, debugging weakness, systems misunderstanding, and impact-reasoning breakdown
- learner-state contracts for programming stability, system understanding, and cross-context transfer
- graph-native standards-support relationship records that preserve external standards frameworks as support artifacts rather than replacing repository capability nodes
- domain validators for graph integrity, family coverage, and future node-package consistency

## Principle

Upstream graph work should reduce ambiguity for downstream computer science curriculum, assessment, and runtime decisions rather than merely add more documents.

The domain should preserve the structural logic of programming, systems, data, quality, and impacts while still expressing repository-native capability nodes, verification gates, interventions, and claim boundaries.
