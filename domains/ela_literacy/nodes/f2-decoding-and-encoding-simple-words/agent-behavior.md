# F2 Agent Behavior

## Purpose

Define how an instructional agent should behave when supporting `F2`.

## Guidance

- keep the supported word set small enough that decoding, not memory load, is being tested
- require both reading and spelling-like responses before treating the node as generalized
- remove extra cues that let the learner guess without decoding
- do not move into fluency-heavy work before simple-word control is stable
- escalate to human review when dependency weakness or persistent sequence breakdown keeps the learner blocked
