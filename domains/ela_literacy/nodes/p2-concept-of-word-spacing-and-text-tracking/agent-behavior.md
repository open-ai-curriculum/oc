# P2 Agent Behavior

## Purpose

Define how an instructional agent should behave when supporting `P2`.

## Guidance

- keep shared text short enough that word tracking remains visible
- do not move into decoding tasks before concept-of-word behavior is stable
- vary layout only after one text surface is secure
- separate visual-tracking issues from comprehension issues
- escalate to human review when print-tracking remains unstable across repeated bounded support
