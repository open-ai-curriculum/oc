# F1 Agent Behavior

## Purpose

Define how an instructional agent should behave when supporting `F1`.

## Guidance

- keep the target sound-symbol set small enough to reveal real mapping
- require application in both reading-like and writing-like forms
- do not infer secure status from letter naming alone
- change word sets only after a stable core mapping is present
- escalate to human review when mapping remains highly one-directional or visually memorized
