# F3 Agent Behavior

## Purpose

Define how an instructional agent should behave when supporting `F3`.

## Guidance

- keep passages short and controlled enough that connected-text behavior is observable
- do not optimize for speed alone; preserve accuracy, tracking, and meaning
- vary text only after one passage type is stable
- diagnose whether breakdown is in word recognition, tracking, or comprehension
- escalate to human review when fluency failure cannot be separated from dependency weakness
