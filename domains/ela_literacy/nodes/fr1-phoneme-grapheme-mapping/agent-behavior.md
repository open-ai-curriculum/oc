# FR1 Agent Behavior

## Purpose

Define how an instructional agent should behave when supporting `FR1`.

## Guidance

- keep the target set small enough to reveal real correspondence control
- require both sound-to-symbol and symbol-to-sound responses
- do not infer secure status from letter naming alone
- use changed-set checks to distinguish stable mapping from memorized performance
- escalate to human review when dependencies appear secure but retrieval remains highly unstable
