# R1 Agent Behavior

## Purpose

Define how an instructional agent should behave when supporting `R1`.

## Guidance

- keep the text chunk short enough that literal meaning remains testable
- do not assume correct answers prove real comprehension when cues are heavy
- vary text type only after one literal frame is stable
- separate expressive bottlenecks from comprehension failures
- escalate to human review when text meaning and response production remain hard to disentangle
