# ELA/Literacy vs Mathematics Comparative Analysis And Implementation Plan

## Purpose

Compare the current `ela_literacy` and `mathematics` domain packages at the level of:

- abstraction
- graph completeness
- repository package completeness
- implementation depth
- operational maturity

Then define the implementation plan required to raise the ELA/literacy spine and graph package to the same quality bar and operational standard already represented by the mathematics domain.

## Governing Comparison Rule

This comparison does not ask whether the two domains are equal in content size.

It asks whether they are operating at the same level of:

- graph-system abstraction
- governed package completeness
- standards-native graph maturity
- runtime and attachment readiness
- review and expansion discipline

## Executive Judgment

### 1. Abstraction Level

Assessment: approximately aligned

Both domains now define:

- a canonical master graph
- node execution contracts
- standards-native overlays
- graph-level architecture
- graph-level runtime and attachment contracts
- export surfaces

At the architecture level, ELA/literacy now sits in the same class of package as mathematics.

### 2. Current Seeded-Spine Depth

Assessment: ELA/literacy is strong and internally complete

Within the currently seeded ELA/literacy spine:

- all `26` current nodes have full operational node packages
- every current node now carries the same operational artifact pattern
- the spine is continuous from pre-K readiness through advanced secondary literacy

At the seeded-spine level, ELA/literacy is no longer shallow.

### 3. Domain-Scale Completeness

Assessment: mathematics remains ahead

Mathematics still has a broader and more mature domain package because it includes:

- a much larger modeled graph (`132` nodes versus `26`)
- more specialized scoped graph views
- more historically accumulated frontier reviews
- more explicit breadth-completeness reasoning at peer-level slices
- more downstream architectural narrative about expansion from current frontier into later territory

ELA/literacy now matches mathematics structurally, but not yet in total modeled breadth or accumulated maturity.

### 4. Operational Maturity

Assessment: partially aligned, not yet fully equal

ELA/literacy now has the same major operational layers as mathematics, but mathematics still has a deeper maturity profile because it already includes more of these as historically sequenced and frontier-specific artifacts rather than first-pass parity documents.

## Concrete Similarities

These areas are no longer the main problem.

### A. Same Graph-System Abstraction

Both domains now treat the graph as:

- canonical topology
- standards backbone
- attachment surface
- learner-state coordinate system
- navigation surface

That is the most important parity achievement.

### B. Same Node-Package Model

Both domains now use node packages with:

- verification models
- failure taxonomies
- learner-state models
- interventions
- observability views
- transition rules
- agent guidance

### C. Same Operational Overlay Pattern

Both domains now have:

- standards-augmented graph artifacts
- artifact attachment registries
- runtime slice registries
- graph export tooling

## Concrete Differences

These are the real remaining deltas.

### 1. Mathematics Has More Scoped Graph Views

Mathematics contains additional graph-view artifacts such as:

- `zero-state-to-f1-spine`
- `zero-state-to-fraction-peer-math-graph`
- branch and peer coverage reviews tied to a named frontier

These are not just extra documents. They create clarity about:

- what counts as a valid prerequisite path
- what counts as adequate peer-level breadth
- what the current frontier really covers

ELA/literacy currently has:

- `prek12-literacy-spine`
- `first-node-target`
- the canonical graph

That is useful, but it is still less analytically differentiated than mathematics.

### 2. Mathematics Has Stronger Frontier-Based Coverage Logic

Mathematics explicitly distinguishes between:

- prerequisite spine sufficiency
- peer-level coverage sufficiency
- post-frontier downstream expansion

ELA/literacy does not yet have the same frontier-language precision.

Current ELA/literacy documents describe the whole spine and the seeded node set, but they do not yet provide the same level of explicit answers to questions such as:

- what must exist before `F1` alphabetic principle is legitimate
- what counts as peer-level breadth around the current literacy-entry frontier
- what downstream composition or analysis branches must exist at comparable maturity before a frontier is considered coherent

### 3. Mathematics Has Greater Breadth Of Implemented Territory

Mathematics has `132` modeled capability nodes across a much broader domain territory.

ELA/literacy has `26` modeled capability nodes. Those nodes are now operationally deep, but they still represent a much more compressed graph.

This does not mean the ELA graph is poor. It means:

- ELA has strong spine depth
- mathematics has stronger total territory coverage

### 4. ELA Standards Work Remains More Scaffold-Oriented

The ELA node standards map is rich, but much of it remains intentionally scaffolded with:

- placeholder crosswalk logic
- review-sensitive identifier posture
- family-level review sequencing
- explicit out-of-scope cases

That is correct governance, but it means the standards overlay is operationally present without being equally hardened.

### 5. ELA Runtime And Attachment Layers Are Still Slice-Seeded

ELA now has:

- one initial runtime slice
- one initial attachment registry

Mathematics has the same pattern, but in context mathematics sits on a larger already-modeled implementation surface and a longer chain of frontier documents.

The ELA versions are therefore structurally correct but earlier in maturity.

## Comparative Conclusion

The domains are now approximately equal in graph-system abstraction and package structure.

They are not yet equal in:

- modeled territory breadth
- frontier-coverage analysis maturity
- standards hardening maturity
- scoped graph-view richness
- operational maturity accumulation

The right target is not to copy mathematics document-for-document. The right target is:

1. preserve the parity already achieved in architecture and node packages
2. raise ELA/literacy in the areas where mathematics is genuinely more mature
3. do so in a way appropriate to literacy rather than forcing mathematics-shaped artifacts where they do not fit

## Implementation Plan

### Phase 1. Add Frontier-Specific ELA Graph Views

Objective:

- give ELA/literacy the same frontier-analysis precision mathematics has

Deliverables:

- `oral-language-to-f1-literacy-spine.md`
- `oral-language-to-f1-literacy-spine.yaml`
- `pre-k-to-sentence-level-literacy-peer-graph.md`
- `pre-k-to-sentence-level-literacy-peer-graph.yaml`
- `composition-and-response-peer-coverage-review.md`

Why this matters:

- distinguishes strict prerequisites from peer-level breadth
- clarifies what makes `F1`, `R2`, `W2`, or `W3` legitimate targets
- gives future expansion a cleaner boundary language

Exit condition:

- ELA/literacy can answer the same class of frontier questions mathematics already answers

### Phase 2. Harden Domain-Specific Coverage Reviews

Objective:

- move from generic parity statements to named coverage judgments

Deliverables:

- `foundational-literacy-peer-coverage-review.md`
- `reading-writing-cross-branch-coverage-review.md`
- `advanced-secondary-literacy-coverage-review.md`

Why this matters:

- mathematics has stronger peer-level coverage reasoning
- ELA needs explicit coverage judgments by frontier, not only by full-spine narrative

Exit condition:

- each major ELA frontier has a documented answer to “is this graph broad enough for the claimed territory?”

### Phase 3. Expand Runtime And Attachment Coverage Beyond The Initial Slice

Objective:

- move ELA operational layers beyond seed-only registries

Deliverables:

- second attachment registry for `F2` through `W3`
- second runtime slice for reading-response and composition
- third attachment/runtime slice for advanced analysis (`R4` through `A2`)

Why this matters:

- mathematics parity is not achieved by one seed registry alone
- ELA needs multiple operational slices covering different graph regions

Exit condition:

- runtime and attachment layers cover early, middle, and advanced literacy slices

### Phase 4. Add ELA-Specific Validation And Generation Checks

Objective:

- give ELA/literacy its own structural inspection layer instead of relying mostly on repository-wide checks

Deliverables:

- `scripts/validate-ela-literacy-domain.js`
- checks for node-package completeness
- checks for generated overlay freshness
- checks for export freshness
- checks for standards-map and graph consistency

Why this matters:

- mathematics maturity is partly operational discipline, not just documents
- ELA still lacks a dedicated domain audit path

Exit condition:

- ELA can be validated as a domain package in one command with explicit failure reporting

### Phase 5. Harden Standards Alignment From Scaffold To Reviewed Targets

Objective:

- move the ELA standards surface from graph-native scaffold to graph-native reviewed alignment

Deliverables:

- reviewed identifier targets for the `F`, `R`, `L`, and `W` families
- preserved explicit out-of-scope handling for pre-standard readiness nodes
- updated standards overlay after reviewed target assignments

Why this matters:

- this is one of the clearest remaining maturity differences from mathematics
- graph-native standards are present, but many ELA alignments remain placeholder-sensitive by design

Exit condition:

- priority ELA families have reviewed node-level identifier posture rather than placeholder-only crosswalk intent

### Phase 6. Expand The ELA Graph Where Compression Is Hiding Real Capability Structure

Objective:

- increase literacy graph fidelity where the current 26-node model is too compressed

Candidate expansion areas:

- deeper foundational reading granularity between `F1`, `F2`, and `F3`
- stronger comprehension branch differentiation between literal, inferential, analytical, and disciplinary reading
- stronger writing branch differentiation between transcription, revision, argument, research, and genre control
- more explicit multilingual/language-access dependent surfaces where governance requires them

Why this matters:

- mathematics has more breadth partly because it models more intermediate capability structure
- ELA may need more nodes, not just more documents, to truly reach comparable domain maturity

Exit condition:

- the ELA graph is expanded where capability compression currently hides meaningful dependency structure

### Phase 7. Produce A Named ELA Comprehensive Graph Architecture Package

Objective:

- bring ELA to the same “comprehensive graph architecture” status as mathematics not only in form but in maturity

Deliverables:

- revised `master-ela-literacy-graph-architecture.md` with richer entity and edge semantics
- ELA-specific artifact edge taxonomy additions where literacy needs differ
- runtime examples spanning early literacy, composition, and advanced-analysis slices
- a domain-level implementation percentage and maturity tracker

Why this matters:

- mathematics architecture is not only present, it is embedded in a fuller operational story
- ELA should reach the same quality by connecting architecture to concrete slice execution and growth metrics

Exit condition:

- the ELA architecture package reads as comprehensive, not merely parallel

## Recommended Sequence

The recommended order is:

1. scoped graph views
2. frontier coverage reviews
3. multi-slice runtime and attachment expansion
4. ELA-specific validator
5. standards hardening
6. graph granularity expansion where needed
7. architecture package refinement and maturity tracker

## Why This Sequence

- scoped view clarity should come before additional expansion
- coverage reviews should come before declaring new territory complete
- runtime and attachment expansion should precede heavy architecture revision so the architecture reflects real operational slices
- standards hardening should happen after the graph and slices are stable enough to support reviewed identifiers
- graph expansion should be justified by explicit fidelity gaps, not by document symmetry alone

"Do the two domain packages now sit at approximately the same abstraction level?"

Answer:

- yes

If the question is:

"Do they now sit at the same completeness and implementation depth?"

Answer:

- not yet

"Is there now a clear implementation path to raise ELA/literacy to that level?"

- yes, and the remaining path is now specific, bounded, and mostly about frontier analysis maturity, standards hardening, validation, and graph-fidelity expansion rather than basic package scaffolding.
