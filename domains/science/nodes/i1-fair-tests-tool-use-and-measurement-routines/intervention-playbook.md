# I1 Intervention Playbook

## Purpose

Map `I1` failure modes to bounded intervention responses.

## Intervention Rules

### For I1-FM-01 Fairness Breakdown

Primary interventions:

- use side-by-side examples of one-change versus many-change tests
- ask which part must stay the same before running the test
- rehearse one-variable language with simple classroom materials

Expected effect:

- learner keeps the test focused on one intended comparison

Reverification target:

- one fair-test item

### For I1-FM-02 Tool-Purpose Mismatch

Primary interventions:

- match common science tools to what each one measures or helps observe
- compare a correct and incorrect tool choice
- practice naming the purpose before picking the tool

Expected effect:

- learner selects and uses tools more appropriately

Reverification target:

- one tool-selection item

### For I1-FM-03 Recording Instability

Primary interventions:

- use a simple table with one row per trial
- model how to record immediately after each observation
- compare complete and incomplete records

Expected effect:

- learner records outcomes consistently enough for comparison

Reverification target:

- one recording task

### For I1-FM-04 Context Boundary

Primary interventions:

- repeat the same fair-test routine across motion, plant, and materials contexts
- preserve the question structure while changing materials
- fade prompts gradually after one successful familiar task

Expected effect:

- learner transfers investigation routines beyond one setting

Reverification target:

- one transfer item

## Escalation Rule

If the same dominant failure mode remains `blocking` after two bounded intervention cycles, escalate to review for language load, executive-load demands, or prompt-design mismatch rather than intensifying the same local practice blindly.

## Human Review Rule

Interventions may be suggested automatically, but the teacher or human reviewer must remain able to override the recommendation when motor access, safety concerns, language access, or context suggests a different explanation of the learner state.
