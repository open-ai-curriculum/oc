# A2 Learner State Model

The learner-state for `A2` tracks:

- dependency security on `A1` and `D4`
- direct accuracy on capstone design, computational integration, and critique tasks
- explanation quality for advanced science-engineering coordination
- transfer performance across unfamiliar capstone contexts
- delayed retention outcome

`secure` requires all verification gates to pass. Any delayed failure or dependency instability should move the learner to `unstable`.
