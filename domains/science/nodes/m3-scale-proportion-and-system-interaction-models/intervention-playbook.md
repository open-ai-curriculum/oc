# M3 Intervention Playbook

## Purpose

Map `M3` failure modes to bounded intervention responses.

## Intervention Rules

### For M3-FM-01 Scale Blindness

Primary interventions:

- compare close-up and system-wide views of the same phenomenon
- ask what can be seen at one scale but not the other
- label models with the scale they represent

Expected effect:

- learner notices why scale matters more reliably

Reverification target:

- one scale item

### For M3-FM-02 Relative-Change Instability

Primary interventions:

- compare bigger/smaller or more/less changes qualitatively
- trace how one relative change affects another part of the system
- use simple comparative frames such as more than, less than, or larger share

Expected effect:

- learner reasons about relative change more reliably

Reverification target:

- one proportion item

### For M3-FM-03 Interaction Omission

Primary interventions:

- map at least two interacting parts and the link between them
- ask what else changes when one part changes
- compare isolated and interaction-aware models

Expected effect:

- learner includes key interactions in the system explanation

Reverification target:

- one interaction item

### For M3-FM-04 Context Boundary

Primary interventions:

- repeat scale-and-interaction routines across life, physical, and Earth/space systems
- preserve the modeling structure while changing the context
- fade scaffolds after one successful familiar example

Expected effect:

- learner transfers scale and system-interaction reasoning across domains

Reverification target:

- one transfer item

## Escalation Rule

If the same dominant failure mode remains `blocking` after two bounded intervention cycles, escalate to review for abstraction load, language load, or prompt-design mismatch rather than intensifying the same local practice blindly.

## Human Review Rule

Interventions may be suggested automatically, but the teacher or human reviewer must remain able to override the recommendation when model complexity, visual access, or response-mode constraints suggest a different explanation of the learner state.
