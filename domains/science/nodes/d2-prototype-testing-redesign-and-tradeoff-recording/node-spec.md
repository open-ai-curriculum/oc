# D2 Node Specification

## Node Identity

- id: `D2`
- name: `prototype_testing_redesign_and_tradeoff_recording`
- domain: `science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `science-bridge-node-package`
- evidence class: `Synthetic`

## Purpose

The learner tests a prototype against design conditions, records what happened, revises the design, and notes when one improvement creates a tradeoff somewhere else.

## Dependencies

- `D1` problem definition, criteria, constraints, and solution sketching
- `I2` variable awareness, data recording, and basic comparison

## Exit Conditions

The learner is secure on `D2` only when all of the following are true:

1. tests a prototype against stated criteria and constraints
2. records what happened clearly enough to compare attempts
3. proposes a redesign based on test evidence
4. notices at least one simple tradeoff or design consequence
5. succeeds across more than one design context and passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- say what happened during a prototype test
- compare an earlier and later version of a design
- explain why a redesign was made
- note that one design choice may help one criterion while making another harder

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
