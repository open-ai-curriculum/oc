# D2 Intervention Playbook

## Purpose

Map `D2` failure modes to bounded intervention responses.

## Intervention Rules

### For D2-FM-01 Test-Result Misread

Primary interventions:

- compare expected versus actual test results
- highlight what counted as success before reading the result
- use simple before-and-after test notes

Expected effect:

- learner interprets test outcomes more accurately

Reverification target:

- one test-result item

### For D2-FM-02 Redesign Disconnect

Primary interventions:

- require the learner to point to the specific test problem before proposing a change
- compare evidence-based and random redesign choices
- use sentence frames such as "I would change this because the test showed..."

Expected effect:

- learner ties redesign choices more directly to evidence

Reverification target:

- one redesign item

### For D2-FM-03 Tradeoff Blindness

Primary interventions:

- compare one design that helps one goal but hurts another
- use a two-column note for "helps" and "makes harder"
- check redesigns against all criteria and constraints one by one

Expected effect:

- learner notices simple tradeoffs more reliably

Reverification target:

- one tradeoff item

### For D2-FM-04 Context Boundary

Primary interventions:

- repeat the test-redesign routine across bridge, shelter, and boat examples
- preserve the redesign logic while changing the design problem
- fade scaffolds after a successful familiar example

Expected effect:

- learner transfers redesign reasoning across early engineering contexts

Reverification target:

- one transfer item

## Escalation Rule

If the same dominant failure mode remains `blocking` after two bounded intervention cycles, escalate to review for executive-load demands, reading load, or prompt-design mismatch rather than intensifying the same local practice blindly.

## Human Review Rule

Interventions may be suggested automatically, but the teacher or human reviewer must remain able to override the recommendation when drawing demands, material familiarity, or response-mode constraints suggest a different explanation of the learner state.
