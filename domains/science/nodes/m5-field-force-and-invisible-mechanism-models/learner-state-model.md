# M5 Learner State Model

The learner-state for `M5` tracks:

- dependency security on `M4` and `I4`
- direct accuracy on field, force, and invisible-mechanism model tasks
- explanation quality for evidence-backed unobservable mechanism reasoning
- transfer performance across advanced scientific systems
- delayed retention outcome

`secure` requires all verification gates to pass. Any delayed failure or dependency instability should move the learner to `unstable`.
