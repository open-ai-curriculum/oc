# M5 Node Specification

## Node Identity

- id: `M5`
- name: `field_force_and_invisible_mechanism_models`
- domain: `science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `science-secondary-node-package`
- evidence class: `Synthetic`

## Purpose

The learner uses evidence-backed models for fields, forces, and other unobservable mechanisms without turning the model into unsupported metaphysical certainty.

## Dependencies

- `M4` conservation, cycle, and energy flow models
- `I4` multivariable investigation, sampling, and error awareness

## Exit Conditions

The learner is secure on `M5` only when all of the following are true:

1. explains how a model can represent an unobservable mechanism from observable effects
2. uses field or interaction models without reducing them to unsupported analogies
3. critiques invisible-mechanism explanations for adequacy and boundary control
4. succeeds across more than one advanced scientific context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `35`

## Observable Evidence

The learner can:

- justify why a field or mechanism model is useful for explaining evidence patterns
- state where the model explains and where it remains simplified or inferential
- compare alternative invisible-mechanism accounts against observed effects
- transfer this reasoning across physics, chemistry, and biology-adjacent cases where appropriate

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
