# D4 Learner State Model

The learner-state for `D4` tracks:

- dependency security on `D3`, `E4`, and `I5`
- direct accuracy on optimization, tradeoff, and design-justification tasks
- explanation quality for evidence-based design reasoning
- transfer performance across advanced engineering contexts
- delayed retention outcome

`secure` requires all verification gates to pass. Any delayed failure or dependency instability should move the learner to `unstable`.
