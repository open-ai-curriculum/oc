# D4 Node Specification

## Node Identity

- id: `D4`
- name: `evidence_driven_optimization_under_constraints`
- domain: `science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `science-secondary-node-package`
- evidence class: `Synthetic`

## Purpose

The learner optimizes designs using evidence, constraint analysis, and tradeoff reasoning without reducing engineering decisions to single-metric preference.

## Dependencies

- `D3` systems design tradeoffs, optimization, and failure analysis
- `E4` quantitative explanation, data critique, and uncertainty reasoning
- `I5` quantitative investigation design, uncertainty, and method critique

## Exit Conditions

The learner is secure on `D4` only when all of the following are true:

1. justifies design changes using evidence tied to explicit criteria and constraints
2. compares tradeoffs across more than one performance dimension
3. critiques optimization claims for uncertainty, measurement adequacy, and unintended consequences
4. succeeds across more than one advanced design context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `42`

## Observable Evidence

The learner can:

- explain why one design is preferable under stated constraints rather than in the abstract
- use evidence to justify optimization choices and revision sequences
- reject claims that a design is best when the metric set or evidence base is too narrow
- compare advanced engineering options using bounded scientific and methodological reasoning

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
