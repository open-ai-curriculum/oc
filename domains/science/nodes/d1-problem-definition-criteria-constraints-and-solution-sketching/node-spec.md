# D1 Node Specification

## Node Identity

- id: `D1`
- name: `problem_definition_criteria_constraints_and_solution_sketching`
- domain: `science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `science-bridge-node-package`
- evidence class: `Synthetic`

## Purpose

The learner identifies the problem to be solved, names criteria and constraints, and sketches a simple solution that fits those problem conditions in an early engineering context.

## Dependencies

- `Q2` question formation, prediction language, and pattern noticing
- `E1` observation-based explanation and evidence talk

## Exit Conditions

The learner is secure on `D1` only when all of the following are true:

1. states the problem in a bounded way
2. identifies at least one success criterion
3. identifies at least one relevant constraint
4. proposes or sketches a simple solution tied to the problem conditions
5. succeeds across more than one design context and passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `21`

## Observable Evidence

The learner can:

- state what the design needs to do
- identify what counts as success
- identify limits such as time, materials, or size
- sketch or describe a solution that fits the stated criteria and constraints

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
