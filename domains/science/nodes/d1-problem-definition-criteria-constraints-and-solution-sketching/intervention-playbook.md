# D1 Intervention Playbook

## Purpose

Map `D1` failure modes to bounded intervention responses.

## Intervention Rules

### For D1-FM-01 Problem Framing Blur

Primary interventions:

- restate the design goal in one sentence
- contrast the problem to solve with extra background details
- ask what the design must help the user or object do

Expected effect:

- learner frames the problem more clearly

Reverification target:

- one problem-definition item

### For D1-FM-02 Criteria-Constraint Confusion

Primary interventions:

- sort statements into success goals versus limits
- label one example of a criterion and one example of a constraint
- compare two design tasks with different material or size limits

Expected effect:

- learner distinguishes criteria from constraints more reliably

Reverification target:

- one criteria-constraints item

### For D1-FM-03 Solution-Fit Breakdown

Primary interventions:

- check each solution against the stated conditions one at a time
- use a simple checklist for criteria and constraints
- compare a fitting and non-fitting design sketch

Expected effect:

- learner proposes solutions that better fit the design conditions

Reverification target:

- one solution-fit item

### For D1-FM-04 Context Boundary

Primary interventions:

- repeat the same design reasoning routine across shelter, bridge, and water tasks
- preserve the criteria-constraints structure while changing the problem
- fade scaffolds after a successful familiar example

Expected effect:

- learner transfers early engineering reasoning across contexts

Reverification target:

- one transfer item

## Escalation Rule

If the same dominant failure mode remains `blocking` after two bounded intervention cycles, escalate to review for language load, drawing demands, or prompt-design mismatch rather than intensifying the same local practice blindly.

## Human Review Rule

Interventions may be suggested automatically, but the teacher or human reviewer must remain able to override the recommendation when response-mode constraints, creativity demands, or local design context suggest a different explanation of the learner state.
