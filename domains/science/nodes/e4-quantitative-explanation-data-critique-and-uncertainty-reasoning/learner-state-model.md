# E4 Learner State Model

The learner-state for `E4` tracks:

- dependency security on `E3` and `I4`
- direct accuracy on quantitative explanation, critique, and uncertainty tasks
- explanation quality for evidence-claim calibration
- transfer performance across disciplinary explanation contexts
- delayed retention outcome

`secure` requires all verification gates to pass. Any delayed failure or dependency instability should move the learner to `unstable`.
