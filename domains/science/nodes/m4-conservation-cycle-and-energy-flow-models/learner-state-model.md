# M4 Learner State Model

The learner-state for `M4` tracks:

- dependency security on `M3` and `E2`
- direct accuracy on conservation, cycle, and energy-flow model tasks
- explanation quality for modeled quantity relationships
- transfer performance across disciplinary systems
- delayed retention outcome

`secure` requires all verification gates to pass. Any delayed failure or dependency instability should move the learner to `unstable`.
