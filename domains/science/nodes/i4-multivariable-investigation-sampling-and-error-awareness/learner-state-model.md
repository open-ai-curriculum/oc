# I4 Learner State Model

The learner-state for `I4` tracks:

- dependency security on `I3`
- direct accuracy on multivariable design, sampling, and error-awareness tasks
- explanation quality for methodological critique
- transfer performance across disciplinary investigation contexts
- delayed retention outcome

`secure` requires all verification gates to pass. Any delayed failure or dependency instability should move the learner to `unstable`.
