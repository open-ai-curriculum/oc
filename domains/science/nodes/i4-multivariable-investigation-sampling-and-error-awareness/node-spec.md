# I4 Node Specification

## Node Identity

- id: `I4`
- name: `multivariable_investigation_sampling_and_error_awareness`
- domain: `science`
- support tier: `Incubation`
- current maturity state: `Drafted`
- supported profile: `science-secondary-node-package`
- evidence class: `Synthetic`

## Purpose

The learner designs and critiques multivariable investigations, uses sampling logic appropriately, and distinguishes likely error sources from unsupported excuses for weak results.

## Dependencies

- `I3` repeated trials, data reliability, and pattern summary

## Exit Conditions

The learner is secure on `I4` only when all of the following are true:

1. identifies relevant variables and controls in multivariable investigations
2. evaluates sampling choices and their consequences for inference
3. distinguishes measurement error, design weakness, and natural variation in bounded cases
4. succeeds across more than one disciplinary investigation context
5. passes a delayed recheck after spacing

## Mastery Criteria

- direct accuracy threshold: `0.90`
- explanation required: `true`
- transfer required: `true`
- retention check days: `35`

## Observable Evidence

The learner can:

- critique investigations that fail to manage important variables
- compare sampling plans using representativeness and scale considerations
- explain how error or uncertainty can change confidence in a result
- justify design revisions using methodological reasoning rather than preference

## Claim Boundary

This node package is an operational draft and not a classroom-release claim.
