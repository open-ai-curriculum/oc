# Science Domain Comparative Analysis And Implementation Plan

## Purpose

Compare the current `science` domain package with the repository's more mature `mathematics` and `ela_literacy` domain packages at the level of:

- abstraction
- graph completeness
- package completeness
- implementation depth
- operational maturity

Then define the implementation plan required to raise the science domain to the same governed quality bar.

## Governing Comparison Rule

This comparison does not ask whether the domains are equal in content size.

It asks whether they are operating at the same level of:

- graph-system abstraction
- governed package completeness
- standards posture
- runtime-readiness potential
- explicit claim boundaries

## Executive Judgment

### 1. Abstraction Level

Assessment: aligned at the scaffold level

The science domain now defines:

- a governed domain README
- a canonical capability-graph scaffold
- a developmental spine
- an upstream build plan
- a first-node-target and wave-priority posture

At the architecture level, science is now inside the same repository model as mathematics and ELA/literacy.

### 2. Current Implementation Depth

Assessment: science is materially earlier

Unlike mathematics and ELA/literacy, science does not yet have:

- a node-package layer
- domain-specific validators
- runtime slices
- artifact-attachment contracts
- graph-level learner-state contracts
- standards-native node mappings

The science domain is therefore structurally present but implementation-light.

### 3. Standards Posture

Assessment: promising but intentionally incomplete

Science benefits from an existing NGSS package, which means the standards family is already codified.

However, the domain does not yet have:

- science-node to NGSS mapping records
- reviewed treatment of performance expectations versus capability nodes
- explicit repository handling for practices, disciplinary core ideas, and crosscutting concepts as linked but distinct entities

### 4. Domain-Scale Modeling

Assessment: coherent spine, not yet a full operational graph

The science scaffold now covers the full K-12 range at the family and wave level. That is enough to preserve architectural direction, but not enough to claim node-level operational maturity.

## Concrete Similarities

These areas are no longer the main problem.

### A. Same Repository Abstraction

Science now uses the same repository logic:

- domain package first
- explicit dependencies
- standards support rather than standards replacement
- claim-boundary discipline

### B. Same K-12 Continuity Rule

Science now preserves one continuous K-12 spine rather than treating elementary, middle, and high school as disconnected silos.

### C. Same Upstream-First Discipline

Science now starts with the earliest meaningful entry points rather than beginning at a later content frontier.

## Concrete Differences

These are the real remaining deltas.

### 1. Science Lacks Node Packages

The mathematics and ELA/literacy domains already encode:

- verification models
- failure taxonomies
- intervention playbooks
- learner-state schemas
- transition rules
- observability guidance

Science does not yet encode any of those at the node level.

### 2. Science Lacks Graph-Level Runtime Contracts

The more mature domains already define domain-level runtime and attachment expectations.

Science still needs:

- graph learner-state and navigation contracts
- artifact-attachment posture
- initial runtime slice definitions

### 3. Science Needs Standards-Structure Hardening

NGSS is structurally more complex than a flat content list. The science domain still needs explicit repository treatment of:

- performance expectations
- science and engineering practices
- disciplinary core ideas
- crosscutting concepts
- grade-band and course-arrangement context

### 4. Science Needs Domain-Specific Failure Taxonomy Design

Science requires explicit differentiation among:

- observation failure
- measurement and method failure
- model confusion
- causal reasoning failure
- evidence-selection failure
- language-access interference
- content-specific misunderstanding

That layer does not yet exist.

## Implementation Plan

### Phase 1. Build The Entry Wave Node Packages

Objective:

- turn the current K-2 entry wave into the first operational science node layer

Deliverables:

- `Q1`, `Q2`, `I1`, `E1`, `L1`, `P1`, and `T1` node packages
- first-pass science verification model patterns
- first-pass science failure taxonomy patterns
- first-pass intervention playbooks and learner-state schemas

### Phase 2. Add Science Graph Contracts

Objective:

- establish domain-level operational contracts before scale-up

Deliverables:

- graph learner-state and navigation contract
- graph artifact-attachment contract
- initial runtime slice schema and example slice

### Phase 3. Add NGSS-Aware Mapping Logic

Objective:

- support standards alignment without collapsing science into standards identifiers

Deliverables:

- science-node standards-map posture document
- explicit relationship model for performance expectations, practices, disciplinary core ideas, and crosscutting concepts
- claim-boundary notes for unresolved mapping questions

### Phase 4. Expand Through Elementary And Middle-Grade Waves

Objective:

- deepen the domain from entry nodes into model-based reasoning and design

Deliverables:

- elementary bridge wave node packages
- middle-grade mechanism and systems wave node packages
- frontier-specific graph views for early science and middle-grade science coherence

### Phase 5. Deepen Advanced Secondary Integration

Objective:

- connect late science reasoning across biology, chemistry, physics, Earth science, and engineering

Deliverables:

- advanced quantitative explanation nodes
- advanced domain-integration nodes
- interdisciplinary synthesis and research-design nodes

## Recommendation

Do not attempt to make science “look mature” by generating a large volume of shallow documents first.

The correct path is:

1. start with the earliest entry wave
2. define strong science-specific verification and failure logic
3. add graph contracts and standards structure
4. then widen the node layer

## Claim Boundary

This plan defines the implementation direction for the science domain. It does not mean the domain is already operationally equivalent to mathematics or ELA/literacy.
