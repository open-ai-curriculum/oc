# Upstream Build Plan

## Purpose

Document the upstream modeling work that should continue strengthening the canonical science graph itself.

## Upstream Targets

- reviewed node-granularity decisions for the first science implementation wave
- science-specific verification-gate patterns that preserve observation, explanation, transfer, and retention
- science-specific failure-mode taxonomies that separate language load, investigation error, model confusion, and content misunderstanding
- learner-state contracts for practice stability, model stability, and cross-domain transfer
- graph-native NGSS relationship records that preserve performance expectations, practices, disciplinary core ideas, and crosscutting concepts as linked but non-identical entities
- domain validators for graph integrity, family coverage, and future node-package consistency

## Principle

Upstream graph work should reduce ambiguity for downstream science curriculum, assessment, and runtime decisions rather than merely add more documents.

The domain should preserve NGSS's structural logic while still expressing repository-native capability nodes, verification gates, interventions, and claim boundaries.
