# Common Core State Standards Overview

## Purpose

This document codifies the Common Core State Standards as a repository standards family with explicit package-level identity, source surface, and claim boundaries.

## Public Source Surface

- Common Core home page: [corestandards.org](https://corestandards.org/)
- Mathematics standards page: [corestandards.org/mathematics-standards/](https://corestandards.org/mathematics-standards/)
- ELA/Literacy standards page: [corestandards.org/english-language-arts-standards/](https://corestandards.org/english-language-arts-standards/)

## Profile Summary

As of the official Common Core site reviewed on April 14, 2026, the current public site indicates that the Common Core State Standards in Mathematics and English Language Arts are temporarily located on the home page while technical challenges are being addressed on CoreStandards.org.

The Common Core standards family includes:

- Common Core State Standards for Mathematics
- Common Core State Standards for English Language Arts
- literacy standards in history/social studies, science, and technical subjects embedded within the ELA/literacy family

## Relationship To Repository Standards Work

The repository already uses a Common Core-aligned mathematics baseline. This package formalizes the broader CCSS family so that:

- mathematics alignment can reference an explicit CCSS package rather than only a repository baseline
- literacy crosswalks can be modeled explicitly for science, technical subjects, or interdisciplinary nodes
- state standards codifications can describe whether they are baseline-aligned, baseline-derived, or structurally distinct relative to CCSS

## Structural Elements To Preserve

The CCSS package should preserve:

- subject family
- grade or grade band
- domain or anchor structure
- cluster or strand grouping where applicable
- code-level identifiers where later added

## Claim Boundary

This document codifies the Common Core standards family for repository use. It does not assert jurisdictional adoption status for any particular state or claim that repository nodes are already mapped to CCSS.
