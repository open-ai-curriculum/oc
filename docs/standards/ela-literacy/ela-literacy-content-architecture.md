# ELA/Literacy Content Architecture

## Purpose

This document codifies the structural content grammar that later ELA/literacy mappings should preserve.

It exists so future node mappings do not flatten literacy into a generic reading label and lose distinctions that matter for claims, interventions, and transfer.

## Structural Layers To Preserve

Future repository ELA/literacy mappings should preserve these layers where applicable:

- anchor standard family
- strand
- grade or grade band
- disciplinary literacy context
- content emphasis or claim note where a node spans multiple strands

## Core Strand Model

The current repository literacy standards package preserves these core strand groupings:

1. foundational reading skills and print concepts
2. reading literature
3. reading informational text
4. writing
5. language
6. speaking and listening
7. disciplinary literacy in history/social studies, science, and technical subjects

## Dependency Interpretation

The repository should not treat these strand labels as isolated silos.

Important dependency rules include:

- speaking and listening often function as upstream access dependencies for reading and writing claims
- language knowledge often mediates both comprehension and written production
- foundational skills constraints can invalidate higher-order comprehension claims if decoding remains unstable
- writing quality claims can be confounded by transcription, sentence control, or idea-generation limits

## Mapping Rule

When a node spans more than one strand, the repository should preserve multi-strand coverage explicitly instead of forcing a single-label classification.

## Claim Boundary

This document codifies the literacy content architecture for repository use. It does not by itself assign standard identifiers to nodes or replace node-level mastery definitions.
