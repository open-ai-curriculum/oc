# ELA/Literacy Performance Framework

## Purpose

This document codifies the package-level proficiency framework for literacy claims so content standards alignment remains distinct from judgments about demonstrated performance.

## Current Performance Bands

The repository uses four literacy performance bands:

1. `Below Basic`
2. `Basic`
3. `Proficient`
4. `Advanced`

## Framework Rules

The package-level framework preserves these rules:

1. performance labels do not replace node-level learner states
2. proficiency claims must be grounded in demonstrated capability rather than exposure or completion
3. literacy performance claims must preserve dependency integrity across oral language, foundational skills, comprehension, language, and writing
4. transfer, retention, and explanation quality remain material to stronger performance claims

## Evidence Considerations

Later literacy mappings should preserve whether a performance judgment is affected by:

- decoding and encoding load
- oral-language access
- comprehension versus word-reading confounds
- transcription constraints
- source use and evidence integration
- transfer across changed texts, prompts, or genres

## Relationship To Node Status

The detailed relationship between learner-state status and performance labels remains defined in:

- [ELA/Literacy Performance Standards Baseline](../ela-literacy-performance-standards-baseline.md)

This package-level artifact exists to make that framework discoverable and versioned within the literacy package itself.

## Claim Boundary

This document codifies the literacy performance framework for repository standards work. It does not establish psychometric equivalence, reporting policy, or reviewed classroom accountability use.
