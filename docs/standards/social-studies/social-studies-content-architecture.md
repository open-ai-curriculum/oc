# Social Studies Content Architecture

## Purpose

This document codifies the structural content grammar that later social studies mappings should preserve.

It exists so future node mappings do not flatten social studies into a generic topic list and lose distinctions that matter for claims, interventions, transfer, and public argument.

## Structural Layers To Preserve

Future repository social studies mappings should preserve these layers where applicable:

- anchor standards family
- disciplinary strand
- grade or grade band
- inquiry or evidence posture
- civic, geographic, economic, historical, or interdisciplinary context
- claim note where a node spans multiple strands

## Core Strand Model

The current repository social studies standards package preserves these core strand groupings:

1. inquiry, evidence, and discourse
2. history and source reasoning
3. civics and collective decision-making
4. geography and human-environment systems
5. economics and exchange systems
6. advanced secondary synthesis and public argument

## Dependency Interpretation

The repository should not treat these strand labels as isolated silos.

Important dependency rules include:

- inquiry and evidence work often function as upstream access dependencies for every disciplinary strand
- history, civics, geography, and economics claims can require cross-strand support when public reasoning or policy analysis is involved
- discourse and public-communication demands can mediate higher-order performance claims across the domain
- language-access issues must remain distinguishable from content-mastery claims

## Mapping Rule

When a node spans more than one strand, the repository should preserve multi-strand coverage explicitly instead of forcing a single-label classification.

## Claim Boundary

This document codifies the social studies content architecture for repository use. It does not by itself assign official external standard identifiers to nodes or replace node-level mastery definitions.
