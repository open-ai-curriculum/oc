# Social Studies Framework Codification Model

## Purpose

This document defines how the repository should codify external social studies standards frameworks before node-to-framework mapping work is treated as parity-grade alignment.

## Why Social Studies Needs A Separate Codification Model

Unlike mathematics, social studies standards are often distributed across multiple overlapping organizational surfaces:

- inquiry practices
- disciplinary strands such as history, civics, geography, and economics
- grade-band or course expectations
- state-specific civics, government, or history requirements

The repository needs an explicit codification method so those surfaces can be represented without collapsing them into one flat list.

## Codification Principles

1. Preserve the repository capability graph as the governing instructional model.
2. Codify public standards structures conservatively and keep claim boundaries explicit.
3. Separate inquiry-process expectations from disciplinary-content expectations when the source framework distinguishes them.
4. Preserve jurisdiction-specific strand structures when they materially affect mapping.
5. Record draft and review-required status whenever crosswalks depend on interpretation rather than explicit identifier-level correspondence.

## Required Framework Sections

Each codified external social studies framework or state profile should contain:

- jurisdiction or framework family
- public source surface
- repository codification status
- relationship to the repository social studies content baseline
- relationship to the repository social studies performance baseline
- inquiry or practice structure when present
- disciplinary strand structure
- grade-band or course-band structure
- notable terminology or organizational divergences
- mapping-readiness notes
- unresolved review questions
- claim boundary

## Required Metadata For Later Mapping

When a profile becomes detailed enough for node mapping, its codified records should support:

- framework or jurisdiction identifier
- grade, grade band, or course band
- inquiry, practice, or process strand when available
- content strand or domain
- anchor statement or expectation summary
- optional official identifier
- baseline-equivalence note
- divergence note
- review status

## Social Studies Profile Types

The repository expects three recurring social studies profile shapes:

1. inquiry-centered frameworks
   - frameworks that foreground investigation, evidence, and argument and then connect those practices to disciplinary content
2. discipline-bundled state standards
   - standards that organize history, civics, geography, and economics under a single public social studies structure
3. course-distinct secondary frameworks
   - standards that split middle- or high-school expectations by course or graduation requirement

## Minimum Useful Codification

A social studies profile is minimally useful when it includes:

- enough structure to identify inquiry and disciplinary strands
- enough grade-band or course-band detail to locate future node mappings
- explicit notes on whether the framework is broad, partial, or course-specific
- explicit repository posture toward identifier-level mapping

A profile is not sufficient for parity-grade mapping if it only states that a framework exists.

## Claim Boundary

This model defines repository structure for social studies standards codification. It does not certify policy compliance, external adoption status, or complete equivalence to official state or national publications.
