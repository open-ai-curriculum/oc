# State Standards Codification Model

## Purpose

This document defines how the repository should codify a publicly distributed state standards set before any node-to-state mapping work begins.

## Codification Principles

1. Use public state standards documents only as source material for codification, not as a substitute for the repository's capability model.
2. Preserve state-specific structure where it materially affects alignment.
3. Record similarity to the canonical baseline, but do not assume equivalence unless reviewed.
4. Keep content standards and performance standards distinct even when a state publishes them together.
5. Mark each state profile as draft until its structure, terminology, and claim boundaries are reviewed.

## Required Sections For A State Standards Profile

Each state standards profile should contain:

- jurisdiction name
- subject area
- publication family or source profile
- repository codification status
- relationship to the canonical mathematics content baseline
- relationship to the repository performance baseline
- grade-band structure
- domain or strand structure
- notable divergences or state-specific terminology
- unresolved review questions
- claim boundary

## Required Metadata For Later Mapping

When a state profile becomes detailed enough for node mapping, the codified records should support:

- jurisdiction code
- grade or grade band
- content strand or domain
- cluster or reporting group when available
- anchor statement
- optional official identifier
- baseline equivalence note
- divergence note
- review status

## State Profile Types

The repository expects three recurring state profile shapes:

1. baseline-aligned profiles
   - states whose published mathematics standards closely follow a canonical multi-state baseline
2. baseline-derived profiles
   - states that retain major shared structures but change identifiers, wording, sequencing, or reporting groups
3. state-distinct profiles
   - states with materially different organization, terminology, or content emphasis

## Minimum Useful Codification

A state profile is minimally useful when it includes:

- public-state identification
- grade-band and domain structure
- enough anchor language to tell where node mappings would later land
- notes about whether the state is close to or far from the canonical baseline

A state profile is not yet sufficient for node mapping if it only says that a public standards document exists.

## Claim Boundary

This model defines repository structure for state standards codification. It does not certify legal adoption status, current policy enforcement, or complete equivalence to official state publications.
