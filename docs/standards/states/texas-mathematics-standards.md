# Texas Mathematics Standards

## Jurisdiction

- jurisdiction: `Texas`
- subject: `Mathematics`
- source profile: `Texas Essential Knowledge and Skills for Mathematics (TEKS)`
- repository codification status: `Drafted`

## Public Source Surface

- TEA mathematics subject page: [tea.texas.gov/academics/subject-areas/mathematics](https://tea.texas.gov/academics/subject-areas/mathematics)
- Mathematics TEKS review page: [tea.texas.gov/.../mathematics-texas-essential-knowledge-and-skills](https://tea.texas.gov/academics/curriculum-standards/teks-review/mathematics-texas-essential-knowledge-and-skills)

## Profile Summary

Texas uses the TEKS framework rather than a direct Common Core structure. The public TEA mathematics surface presents state standards as what students should know and be able to do, with explicit student expectations and a long-standing course and grade structure.

This makes Texas a state-distinct profile for repository purposes, even though much of the mathematical substance overlaps the canonical baseline.

## Relationship To Canonical Mathematics Content Baseline

- relationship type: `state-distinct`
- core structure: `grade-level and course-level TEKS with state-specific organization`
- notable state distinction: `student expectations and process-oriented expectations are integral to the TEKS surface and should not be flattened into generic CCSS-style domain labels`

## Relationship To Repository Performance Baseline

Texas content standards and curricular expectations should remain separate from repository proficiency bands. If later reporting or assessment-level descriptors are added, they should be codified as a separate Texas performance artifact rather than merged into the TEKS content profile.

## Grade-Band Structure

### Kindergarten Through Grade 8

Primary organizational pattern:

- grade-level mathematics TEKS
- strands and student expectations within grade
- explicit mathematical process expectations carried across content work

Operational fit with repository node work:

- strong fit for early number, operations, measurement, geometry, and data nodes
- later node mappings should preserve both content expectation and any relevant process expectation linkage

### High School

Primary organizational pattern:

- course-based mathematics standards
- public TEA surface includes Algebra I, Geometry, Algebra II, and additional mathematics pathways

Operational fit with repository node work:

- algebra, functions, geometry, statistics, and advanced secondary nodes may require course-aware mapping
- a single node may need multiple Texas references when its capability spans more than one course expectation

## Domain Or Strand Structure

Texas uses a strand-and-expectation style organization rather than the exact Common Core domain and cluster pattern.

Repository crosswalk work should expect to represent:

- grade or course
- strand
- student expectation
- optional process-standard linkage

## Notable Divergences Or State-Specific Terminology

- `TEKS` is the governing state standards family
- `student expectations` are central mapping units
- `mathematical process standards` are part of the state structure and should be preserved as explicit metadata

## Mapping Readiness Notes

Texas is important to codify early because it is structurally different from a baseline-aligned CCSS-style profile. Building this profile early will force the repository mapping model to support more than one external standards grammar.

## Unresolved Review Questions

- how to represent TEKS process expectations alongside content expectations in node mappings
- whether Texas mappings should attach at the strand level, student-expectation level, or both
- which additional secondary-course surfaces should be treated as first-class mapping targets in the initial repository phase

## Claim Boundary

This document codifies the Texas public mathematics standards surface for repository alignment planning. It does not claim legal sufficiency for Texas implementation or complete correspondence between repository nodes and TEKS expectations.
