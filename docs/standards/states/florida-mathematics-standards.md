## Profile Summary

Florida uses the B.E.S.T. mathematics standards and publishes a distinct set of Mathematical Thinking and Reasoning standards alongside the content standards.

For repository purposes, Florida is a baseline-derived profile: the content scope overlaps the canonical mathematics baseline, but the state uses a distinct benchmark-and-reasoning structure that should be modeled directly.

## Relationship To Canonical Mathematics Content Baseline

- relationship type: `baseline-derived`
- core structure: `grade and course mathematics benchmarks with state-specific benchmark organization`
- notable state distinction: `Mathematical Thinking and Reasoning standards operate alongside content benchmarks and should be preserved in the repository model`

## Relationship To Repository Performance Baseline

Florida publishes statewide assessments aligned to the B.E.S.T. standards. Even so, repository performance alignment should remain a separate layer from content-benchmark alignment. If Florida achievement-level or reporting descriptors are later codified, they should be documented as a Florida performance profile, not merged into the benchmark taxonomy.

## Grade-Band Structure

### Elementary And Middle Grades

Primary organizational pattern:

- grade-level benchmarks
- explicit state mathematics benchmark structure
- mathematical thinking and reasoning expectations that cut across content work

Operational fit with repository node work:

- suitable for early number, operations, fractions, measurement, geometry, and data nodes
- node mappings should preserve both content benchmark references and any relevant reasoning-standards linkages

### High School

Primary organizational pattern:

- course-linked mathematics standards and assessments
- Algebra 1 and Geometry are especially important public alignment surfaces because the state assessment system names them directly

Operational fit with repository node work:

- algebra and geometry nodes can map relatively directly
- broader function, modeling, or statistics nodes may require more than one benchmark or course reference

## Domain Or Strand Structure

Florida's repository codification should distinguish:

- content benchmarks
- Mathematical Thinking and Reasoning standards
- grade or course
- assessment-facing surfaces where they materially affect mapping decisions

## Notable Divergences Or State-Specific Terminology

- `B.E.S.T.` is the governing state content family
- `Mathematical Thinking and Reasoning` is a first-class standards surface, not just advisory language
- `FAST` and mathematics EOC assessments are alignment-relevant but should remain separate from content taxonomy

## Mapping Readiness Notes

Florida is a useful early profile because it tests whether the repository can represent content expectations plus cross-cutting mathematical reasoning expectations without collapsing one into the other.

## Unresolved Review Questions

- whether Mathematical Thinking and Reasoning standards should be modeled as secondary mappings on each node or as a parallel alignment layer
- how tightly later node mappings should bind to assessment-facing surfaces such as Algebra 1 and Geometry EOCs
- whether a separate Florida performance-standards profile should be codified from statewide achievement-level materials

## Claim Boundary

This document codifies Florida's public mathematics standards surface for repository alignment planning. It does not claim complete equivalence between repository nodes and Florida benchmarks or statewide assessment expectations.
