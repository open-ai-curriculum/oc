# Accessibility Review Checklist

## Purpose

This checklist provides a minimum review pass for curriculum materials before publication.

It should be used alongside:

- `accessibility-standards.md`
- the project's broader quality review framework

## How to Use

Reviewers should mark each item as:

- pass
- needs revision
- not applicable

If any essential item fails, the artifact should not be approved for release.

## Core Checklist

### Structure

- The document uses real headings and logical section order.
- Lists, tables, and labels are used semantically rather than only visually.
- Link text is descriptive.
- The reading order is logical when read top to bottom.

### Text

- The writing is clear enough for the intended audience.
- Specialized terms are explained when needed.
- Instructions are explicit and not unnecessarily dense.
- Essential content is not hidden only in footnotes, callouts, or visuals.

### Visual Content

- Informative images include alt text or equivalent explanation.
- Complex diagrams, charts, or visuals include adjacent explanatory text.
- Color is not the only way meaning is conveyed.
- Visual contrast is adequate for readability where color is used.

### Multimedia

- Video includes captions.
- Audio includes a transcript or equivalent text alternative.
- Essential visual-only information is described in text or narration.

### Tables and Data

- Tables have clear headers.
- Charts have labels and text explanation.
- Data interpretation does not depend only on color or position.

### Interaction

If the artifact is interactive:

- keyboard-only use is possible
- focus state is visible
- controls are labeled clearly
- instructions for completion are explicit

### Instructional Flexibility

- The resource provides more than one reasonable way to access key content, where appropriate.
- The resource provides more than one reasonable way for students to show understanding, where appropriate.
- Avoidable sensory, language, or motor barriers have been reduced.

### Assessment Accessibility

If the artifact includes assessment:

- directions are clear and concise
- reading load is appropriate to the standard being measured
- diagrams and prompts are accessible
- the format does not create unnecessary barriers unrelated to the learning target

### Format and Distribution

- The source format is structurally editable, not only a final PDF or image.
- A printable or low-bandwidth path exists if the resource is intended for wide reuse.
- The release format does not depend on a locked platform for basic use.

## Required Release Threshold

An artifact should not be released if:

- essential structure is non-semantic
- required text alternatives are missing
- core instructions are inaccessible
- essential multimedia lacks captions or transcripts
- the format prevents basic access for intended users
