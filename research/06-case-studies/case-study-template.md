# Case Study Template

## Purpose

Use this template when documenting open or open-adjacent curriculum initiatives relevant to this repository.

The goal is to capture operationally useful evidence rather than marketing summary.

## Required Sections

### 1. Overview

- organization
- initiative name
- date range or current status
- curriculum area and grade span

### 2. Openness Model

- openly licensed
- free to access but restricted
- open-adjacent nonprofit model
- mixed model

Include the specific license or terms if known.

### 3. Delivery Model

- web
- downloadable files
- print
- LMS-compatible distribution
- tool or platform dependencies

### 4. Quality Model

- standards alignment method
- review process
- teacher supports
- assessment approach
- accessibility considerations

### 5. Governance and Sustainability

- who maintains the materials
- how updates happen
- how contributors participate
- how the work is funded or sustained

### 6. Evidence and Outcomes

- adoption or reach evidence
- implementation signals
- strengths supported by evidence
- limits or open questions

### 7. Relevance to This Repository

- what this project should emulate
- what this project should avoid
- what remains unclear

### 8. Sources

List only sources actually used.

## Contribution Rules

1. Distinguish verified facts from inference.
2. Do not label a model fully open if access, reuse, or adaptation is materially restricted.
3. Prefer official sources over third-party commentary.
