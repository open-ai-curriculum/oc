# Licensing Analysis for Open Curriculum

## Purpose

This document records the repository's earlier licensing analysis and compares several licensing approaches that were considered before the repository license was changed to PolyForm Noncommercial 1.0.0.

The immediate question is not whether the project should remain open. It should. The question is which legal tool best supports:

- broad reuse
- global clarity
- contributor expectations
- adaptation and redistribution
- interoperability with the wider OER ecosystem

## Current Repository Position

The active repository license is now PolyForm Noncommercial 1.0.0.

Historically, the repository charter stated that the project and all contents were released into the public domain under the Unlicense. The analysis below is preserved as comparative background from that earlier phase.

That position optimizes for maximal permissiveness, but it also raises practical questions the research base should address explicitly:

- whether contributors understand the rights they are granting
- whether the tool is the clearest fit for educational content rather than software
- how well the approach translates across jurisdictions
- whether institutions expecting familiar OER licensing patterns will find it legible

## Core Licensing Options

### Option 1: Unlicense

Strengths:

- very permissive
- simple intent
- aligns with a no-restrictions philosophy

Risks or tradeoffs:

- more common in software than in educational content
- less familiar to many OER practitioners than CC tools
- may create ambiguity for institutions expecting Creative Commons norms

### Option 2: CC0

Strengths:

- explicitly designed as a public-domain dedication tool
- intended to work globally to the fullest extent possible
- includes a fallback permission layer where full dedication is not effective
- highly legible to open culture and OER communities

Risks or tradeoffs:

- like other public-domain approaches, it does not require attribution
- institutions that prefer an attribution norm may see this as too permissive

### Option 3: CC BY 4.0

Strengths:

- widely recognized in OER
- preserves openness while requiring attribution
- familiar to schools, nonprofits, higher-ed institutions, and repositories

Risks or tradeoffs:

- adds compliance obligations for reuse
- slightly increases reuse friction
- is not equivalent to public domain

### Option 4: More restrictive CC licenses

Examples:

- CC BY-SA
- CC BY-NC
- CC BY-NC-SA

Assessment:

These are usually a weaker fit for this repository's stated mission because they impose additional restrictions that reduce frictionless reuse, especially for curriculum localization, redistribution, and institutional adoption.

## Analysis Against Project Goals

### Goal 1: Maximum Reuse

Best fit:

- CC0 or another robust public-domain-style dedication

Why:

- the project's charter explicitly prioritizes unrestricted reuse, modification, and commercialization

### Goal 2: Global Legibility

Best fit:

- CC0

Why:

- Creative Commons tools are widely recognized in education and OER contexts
- CC0 is explicitly designed for public-domain-style release across jurisdictions

### Goal 3: Ecosystem Compatibility

Best fit:

- CC0 or CC BY

Why:

- both are much more familiar in OER workflows than the Unlicense

### Goal 4: Contributor Clarity

Best fit:

- CC0 or CC BY, depending on whether attribution is required

Why:

- contributors and institutions usually understand Creative Commons signals faster than software-oriented public-domain tools

## Historical Recommendation

Before the repository adopted PolyForm Noncommercial 1.0.0, the strongest public-domain-style candidate identified in this analysis was:

- CC0 1.0 Universal

If the project had wanted attribution while remaining broadly open, the main alternative identified was:

- CC BY 4.0

## Important Distinction: Legal Tool vs Community Norm

The project can choose a highly permissive legal tool while still encouraging community norms such as:

- voluntary attribution
- change logs
- preservation of provenance
- contributor acknowledgment

Those norms should not be confused with legal requirements.

## Recommended Decision Process

Before changing the root licensing statement, the project should answer:

1. Is the project optimizing for pure public-domain release or for broad open reuse with attribution?
2. Will the repository accept external contributions from institutions that expect Creative Commons licensing?
3. Does the project want a licensing tool recognized immediately by the OER community?
4. Will all published assets, including templates and media, use the same licensing model?
