# Mathematics Assessment Guidance

## Purpose

This document defines math-specific assessment expectations for curriculum developed in the repository.

## Core Position

Math assessment should produce evidence of mathematical understanding, not only answer production.

## Assessment Priorities

### 1. Align to the mathematical target

Each assessment task should make clear whether it is intended to assess:

- concept understanding
- procedural fluency
- application
- reasoning and justification

### 2. Include evidence of reasoning when appropriate

Math assessment should often ask students to:

- show work
- explain a method
- compare strategies
- identify or correct an error

### 3. Reduce answer-only fragility

Because AI can generate final answers quickly, math assessment should prefer:

- explanation-rich prompts
- tasks with intermediate reasoning
- tasks that require interpretation or critique

### 4. Protect accessibility without changing the construct

Assessment accessibility should reduce avoidable barriers without changing the mathematics being measured.

## Review Questions

Before release, reviewers should ask:

1. What mathematical understanding is this item supposed to measure?
2. Could a correct final answer still hide misunderstanding?
3. Would explanation or representation reveal more meaningful evidence?
4. Are visuals, notation, and wording accessible for the intended learners?

## Common Risks

- answer-only items used where reasoning evidence is needed
- inaccessible diagrams or notation
- reading load that interferes with the mathematical target
- AI-generated distractors or worked examples that are mathematically unsound
