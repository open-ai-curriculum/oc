# Assessment Strategies

## Purpose

This document summarizes the assessment principles most relevant to the repository's curriculum development work.

It focuses on assessment quality, alignment, accessibility, and integrity rather than on any one platform or product.

## Core Position

Assessment should be treated as evidence design, not only grading infrastructure.

For this repository, strong assessment work should be:

- aligned to standards and objectives
- valid for the intended claim
- accessible to diverse learners
- resistant to shallow completion
- reviewable by educators

## High-Confidence Principles

### 1. Plan assessment with the learning target

Assessment should be designed alongside outcomes and instruction, not appended after the fact.

### 2. Use multiple forms of evidence where appropriate

Strong curriculum should often include:

- formative checks
- performance tasks
- reflection or explanation
- summative evidence where needed

### 3. Review validity, not only convenience

The key question is not whether an assessment is easy to score. It is whether it produces meaningful evidence of the intended learning.

### 4. Design for accessibility and fairness

Assessment should not introduce avoidable barriers unrelated to the learning target.

### 5. Account for AI-enabled misuse

Assessment design should increasingly prefer tasks that require:

- reasoning
- evidence use
- process visibility
- authentic performance

## Practical Assessment Types

Useful patterns for this repository include:

- formative checks for understanding
- project- or performance-based tasks
- portfolio evidence
- standards-aligned rubrics
- self- and peer-reflection when instructionally appropriate

## Assessment Review Questions

Before release, reviewers should ask:

1. What is this assessment intended to show?
2. Does it align with the stated objective or standard?
3. Would success on this task actually indicate understanding?
4. Are there accessibility barriers unrelated to the target?
5. Is the task too easy to complete superficially with generic AI assistance?

## Platform and Tool Implications

The repository should avoid treating assessment quality as equivalent to tool sophistication.

Portable, reviewable, standards-aligned assessment design matters more than platform feature count.
