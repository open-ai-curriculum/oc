# Release Review Checklist

## Purpose

Use this checklist before approving a curriculum artifact for broader repository use or publication.

## Artifact Information

- Artifact title:
- Artifact type:
- Subject:
- Grade / grade band:
- Reviewer:
- Date:

## Core Review Gates

### 1. Alignment and Coherence

- [ ] Objectives are clear
- [ ] Standards claims are plausible
- [ ] Instruction, tasks, and assessment align
- [ ] The artifact is complete enough for release

### 2. Quality and Accuracy

- [ ] Content is factually accurate
- [ ] Subject-specific quality expectations have been reviewed
- [ ] Examples, worked solutions, or sources are trustworthy

### 3. Accessibility and Inclusion

- [ ] Accessibility review has been completed
- [ ] Avoidable barriers have been reduced
- [ ] Inclusive framing and examples have been reviewed

### 4. Assessment Quality

- [ ] Assessment evidence matches the intended learning
- [ ] Tasks are not trivially shallow or gameable
- [ ] Rubrics or criteria are usable where needed

### 5. AI Governance

- [ ] AI use is documented where applicable
- [ ] Required human review was completed
- [ ] Sensitive-topic review was completed where needed

### 6. Privacy and Tooling

- [ ] No real student PII is included
- [ ] Any tool-dependent recommendations have been vetted appropriately
- [ ] No unsafe external workflow is implied

- [ ] Approved
- [ ] Approved with revisions
- [ ] Return to draft
