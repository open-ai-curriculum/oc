# Kindergarten Commercial-Grade Remediation Plan

## Objective

Move the repository's kindergarten offering from a draft integrated program prototype to a market-comparable, adoption-ready instructional system.

## Planning Premise

The current assembled kindergarten textbook and workbook should be treated as proof-of-concept artifacts, not as the direct foundation for a single final commercial-style omnibus book.

The most defensible path is:

1. define the target commercial product architecture
2. build subject-program source products
3. generate teacher-use and student-use assets at lesson scale
4. assemble grade-level reference volumes from those richer sources
5. only then pursue final production-quality books

## Target Product Architecture

### Canonical source products

The repository should treat these as primary products for kindergarten:

- Kindergarten ELA program
- Kindergarten Math program
- Kindergarten Science program
- Kindergarten Social Studies program
- Kindergarten Arts program
- Kindergarten Health and Wellness program
- Kindergarten Movement and PE program
- Kindergarten Technology and Digital Citizenship program

### Required artifact stack for each subject program

- year scope and sequence
- unit map
- lesson map
- student-facing lesson pages
- teacher lesson narratives
- classroom task sheets
- home practice pages where appropriate
- observation tools
- formative assessments
- unit or module assessments where appropriate
- answer keys and exemplar responses
- differentiation and multilingual supports
- family communication assets
- reproducibles and center materials

### Assembled grade-level products

These can remain important, but should be downstream assemblies:

- omnibus kindergarten textbook
- omnibus kindergarten workbook
- omnibus kindergarten teacher edition
- omnibus kindergarten grading key
- omnibus kindergarten assessment pack

## Workstreams

### Workstream 1. Market-normal product definition

Deliverables:

- kindergarten product architecture decision memo
- subject-by-subject component inventory
- page-count and lesson-count targets by subject
- explicit `commercial-grade` definition for this repo

Key decisions:

- whether ELA and math become standalone core programs first
- whether science/social studies/health/arts/PE/technology are lighter module families rather than textbook-heavy programs
- whether omnibus books are canonical or derivative

Exit criteria:

- the repository has numeric targets for lessons, units, pages, and assessments per subject
- the repository stops using `textbook` to describe chapter summaries

### Workstream 2. Kindergarten full-year scope design

Deliverables:

- a full-year kindergarten master scope and sequence
- subject calendars or pacing bands
- cumulative standards map
- explicit articulation of how literacy, math, science, and content subjects interleave

Required depth:

- lesson counts by week
- module lengths
- spiral review points
- benchmark windows
- family reporting windows

Exit criteria:

- every kindergarten week has planned instructional coverage
- every standards claim is traceable to a defined lesson or assessment opportunity

### Workstream 3. ELA commercial-grade build

Reason:

ELA is the strongest existing repository area and should become the first commercial-grade subject prototype.

Deliverables:

- a real kindergarten literacy scope with modules, lesson counts, and text sets
- decodable/read-aloud/shared-reading architecture
- student lesson pages with illustrations and response space
- teacher edition with modeled dialogue, differentiation, and sample responses
- assessment system with observational and performance evidence

Minimum commercial-grade indicators:

- enough lesson material for the full year
- explicit text list and rights/provenance strategy
- repeated page structures and routines
- multilingual and accessibility supports embedded, not tacked on

Exit criteria:

- the kindergarten ELA set can stand on its own as a full-year subject program

### Workstream 4. Math commercial-grade build

Deliverables:

- full-year math scope and sequence
- daily lesson architecture
- manipulatives guidance
- student practice pages and centers
- teacher problem strings, misconceptions, and intervention guidance
- unit assessments and progress-monitoring tools

Required improvements over current state:

- far more lesson volume
- clearer progression from counting to operations, teen numbers, shapes, and measurement
- stronger mathematical representations and discussion prompts

Exit criteria:

- math has a full-year lesson inventory and fully authored student and teacher materials

### Workstream 5. Content-subject build

Subjects:

- science
- social studies
- arts
- health
- PE
- technology

Approach:

- do not force every subject into a textbook-heavy model if the market norm is lighter print plus teacher-led activity
- define the right package shape for each subject

Example expectation:

- science: inquiry-rich modules with observation journals, visual texts, and task cards
- social studies: short topical readers or periodical-style sets, primary source picture supports, and teacher-led discussion guides
- arts/PE/health/technology: teacher-driven lesson cards, student prompt sheets, checklists, and take-home family connections

Exit criteria:

- each non-literacy subject has a clearly justified product form and enough lesson volume to constitute a year plan

### Workstream 6. Student page authoring and design system

Deliverables:

- page templates by artifact type
- illustration and icon system
- response-space standards
- reproducible specifications
- print-production rules for trim, margins, contrast, and accessibility

Critical requirement:

Prompt descriptions must be replaced with actual classroom-ready pages.

Examples of what must exist:

- traceable student worksheets
- cut-and-sort pages
- picture cards
- teacher-record forms
- center cards
- visual schedules
- family take-home sheets

Exit criteria:

- workbook pages are real printable masters, not prose descriptions of pages
- textbook pages are designed instructional pages, not chapter notes

### Workstream 7. Teacher-edition and implementation buildout

Deliverables:

- lesson narrative sets
- pacing and prep guidance
- material lists
- sample teacher language
- misconceptions and intervention notes
- multilingual learner supports
- accessibility accommodations
- classroom management and transition suggestions
- family communication templates

Exit criteria:

- a teacher can teach from the repository without building major missing components independently

### Workstream 8. Assessment, grading, and reporting system

Deliverables:

- observational checklists
- formative checks
- unit or module assessments
- scoring guides
- exemplar responses
- benchmark guidance where appropriate
- gradebook and reporting model

Exit criteria:

- assessment is systematic across the year
- grading guidance is calibrated and consistent

### Workstream 9. QA and market-readiness review

Deliverables:

- executed editorial QA
- executed subject QA
- executed accessibility QA
- executed implementation/usability QA
- executed standards-correlation review

External benchmark lenses:

- EdReports-style usability and alignment expectations
- district adoption rubric expectations around completeness, component integration, and teacher resources

Exit criteria:

- the team can defend the completeness and coherence of the program to an adoption-review audience

## Sequenced Execution Plan

### Phase 1. Reset the target

Time focus:

- 1 to 2 weeks of planning work

Actions:

- define `commercial-grade` for this repository
- decide canonical product architecture
- set lesson-count and page-count targets
- publish kindergarten full-year scope assumptions

### Phase 2. Build the first commercially credible subject

Time focus:

- 4 to 8 weeks minimum for one subject if done seriously

Actions:

- choose ELA first
- expand packet-level literacy work into a true full-year subject program
- create real student pages, teacher edition material, assessments, and reproducibles

Reason:

If one subject cannot reach commercial-grade depth, the omnibus grade-level books will remain superficial.

### Phase 3. Build math to the same level

Actions:

- repeat the ELA pattern for math
- align pacing to the cross-grade K-1 articulation

### Phase 4. Build right-sized non-literacy subject programs

Actions:

- define lighter or heavier package models by subject
- author enough lesson and asset volume for a year plan

### Phase 5. Reassemble omnibus books

Actions:

- rebuild the omnibus textbook, workbook, teacher edition, grading key, and assessment pack from the richer subject systems
- regenerate print-ready outputs from page-stable sources

### Phase 6. Execute real QA and revision

Actions:

- run the QA packets on actual print-ready files
- fix issues
- repeat until the volumes are internally defensible

## Immediate Next Actions

1. Write a kindergarten commercial product architecture memo.
2. Set numeric targets for kindergarten by subject:
   - units
   - lessons
   - student pages
   - teacher-edition pages
   - assessments
   - reproducibles
3. Create a full-year kindergarten master scope and sequence.
4. Promote kindergarten ELA to the first true commercial-grade subject build.
5. Redefine the current omnibus textbook and workbook as interim reference compilations until the richer source programs exist.

## Definition of Done

The kindergarten offering can be called commercially credible only when:

- every subject has a full-year instructional plan
- ELA and math have lesson-level student and teacher materials across the year
- non-literacy subjects have justified and complete year-long package forms
- workbook and textbook pages are fully authored printable masters
- teacher editions contain enough support for a teacher to teach without filling major gaps alone
- assessment, scoring, and reporting systems are complete
- print-ready files have passed editorial, subject, and accessibility QA

Until then, the correct language is still `draft program` or `prototype program`, not `commercial-grade textbook program`.
